## 一、什么是自监督表征学习

**目标**：用**没有人工标注**的数据，训练出一个编码器（Encoder），让它把原始输入（图片、视频、音频）压缩成一个有意义的向量——这个向量应该捕捉数据里真正重要的结构，而不只是记住像素值。

"表征"（Representation）就是这个压缩后的向量，可以理解为"模型对输入的理解"。

**为什么叫"自监督"**：监督信号不来自人打的标签，而来自**数据本身的结构**：
- 图片有空间结构（同一张图的不同区域同属一个场景）
- 视频有时序结构（前一帧和后一帧是连续演变）
- 文本有语言结构（前面的词预测后面的词）

不同的 SSL 方法，本质上是在用不同方式**利用数据自身结构来构造训练信号**。

**为什么重要**：人工标注的数据量有限，互联网上的无标注数据（图片、视频）近乎无限。从无标注数据里学到好的表征，能让下游任务（分类、检测、机器人控制）以很少的标注数据达到很高的性能。VLA 中广泛使用的 SigLIP、CLIP、DINO 编码器，全部是 SSL 训练出来的。

## 二、图片 SSL 需要专门设计，文本不需要

文本领域的自监督任务天然存在且简单：

```
预测下一个词：  "今天天气___"  →  "很好"
掩码填充：     "今天[MASK]很好"  →  "天气"
```

文本是**离散的**（每个位置是一个词），预测目标明确，相邻词之间的语义依存关系本身就是丰富的监督信号。GPT 系列全部基于"预测下一个 token"这一个任务。

**图片没有这个天然优势**。图片是连续的高维数据（一张图 = 数百万个连续像素值），"预测下一个像素"几乎没有意义——相邻像素几乎完全一样，预测太容易，根本学不到语义。

因此视觉领域研究者必须人工设计自监督任务，催生了以下几条技术路线。

## 三、生成式 SSL：MAE（掩码自编码器）

**MAE（Masked Autoencoders，He et al., 2021）** 是把 BERT 的掩码填充思想迁移到图像上最成功的工作。

### 3.1 核心做法

```
原始图像（224×224）
    ↓ 切成 196 个 patch（14×14 grid）
随机遮盖 75% 的 patch（只保留 49 个可见 patch）
    ↓
编码器（ViT-Large）
  只处理可见的 49 个 patch → 得到 49 个 token 的表示
    ↓
解码器（轻量 Transformer）
  输入：49 个编码 token + 147 个可学习的"遮盖 token"
  输出：重建所有 196 个 patch 的像素值
    ↓
训练目标：最小化被遮盖区域的像素重建误差（MSE）
```

**关键设计选择：**
- **75% 的遮盖率**（远高于 BERT 的 15%）：遮盖率越高，任务越难，被迫学习全局语义而不只是局部插值
- **编码器只看可见 patch**：不处理遮盖 token，大幅节省计算量（有效序列长度是原来的 1/4）
- **解码器只在训练时使用**：下游微调时只用编码器，解码器扔掉

### 3.2 为什么高遮盖率有效

低遮盖率（如 15%）时，模型可以用局部插值恢复被遮盖的 patch——看看周围像素，猜出这里大概是什么颜色，不需要理解全局语义。

高遮盖率（75%）时，局部插值基本失效，模型必须建立对整个场景的语义理解才能合理重建——"这里是脸，所以被遮住的这块应该是眼睛"。

### 3.3 MAE vs BERT 的本质区别

| | BERT（文本） | MAE（图像） |
|--|-------------|------------|
| **遮盖单元** | token（离散词） | patch（连续像素块） |
| **预测目标** | 词汇表上的概率分布 | 连续像素值（回归） |
| **遮盖率** | 15% | 75% |
| **为什么差异大** | 文本语义密度高，15% 已经够难 | 像素语义密度低，需要 75% 才够难 |

MAE 的表征在语义分类任务上表现优秀，但在视觉问答和细粒度理解上不如 CLIP（因为没有语言对齐）。

## 四、对比学习：SimCLR / MoCo

**代表**：SimCLR（Google，2020）、MoCo（Meta，2020）

### 4.1 核心思想

同一张图片的两个"变形版本"应该表示相似，不同图片应该表示不相似。

```
原始图片 X
    ├─ 随机变换1（随机裁剪 + 翻转 + 改颜色）→ X₁ ─┐
    └─ 随机变换2（另一套随机变换）         → X₂ ─┘ 正样本对，应该相似

另一张图片 Y → Y₁    X₁ 与 Y₁ 是负样本对，应该不相似
```

**模型结构**：

```
X₁ → 编码器（共享权重）→ 特征 → 投影头 → z₁
X₂ → 编码器（同上）   → 特征 → 投影头 → z₂

损失 = 拉近(z₁, z₂) + 推远(z₁, 所有其他图片的表示)
```

### 4.2 一步训练的完整过程

以 batch 中有 **N=3 张**原始图片为例，走一遍完整的训练步骤。

**第一步：构造 2N 个增强视图**

```
原图 A  →  增强视图 A₁（裁剪+翻转）  /  增强视图 A₂（另一套随机变换）
原图 B  →  增强视图 B₁  /  增强视图 B₂
原图 C  →  增强视图 C₁  /  增强视图 C₂

共 2×3 = 6 个视图送入编码器
```

**第二步：编码器 + 投影头，得到 2N 个向量**

6 个视图全部通过**同一个编码器**（共享权重），再通过**投影头**（2层MLP）：

```
A₁ → 编码器 → 投影头 → z_{A1}（向量，如 128 维）
A₂ → 编码器 → 投影头 → z_{A2}
B₁ → 编码器 → 投影头 → z_{B1}
B₂ → 编码器 → 投影头 → z_{B2}
C₁ → 编码器 → 投影头 → z_{C1}
C₂ → 编码器 → 投影头 → z_{C2}
```

所有向量做**L2 归一化**（让每个向量的长度为 1），之后两个向量的点积就等于余弦相似度。

**第三步：计算 2N×2N 相似度矩阵**

6 个向量两两计算余弦相似度，得到一个 6×6 的矩阵：

```
         z_{A1}  z_{A2}  z_{B1}  z_{B2}  z_{C1}  z_{C2}
z_{A1}  [  1.0    0.87   -0.12    0.05   -0.23    0.11 ]
z_{A2}  [ 0.87    1.0     0.08   -0.15    0.03   -0.09 ]
z_{B1}  [-0.12   0.08    1.0     0.91    0.14   -0.18 ]
z_{B2}  [ 0.05  -0.15    0.91    1.0    -0.07    0.21 ]
z_{C1}  [-0.23   0.03    0.14   -0.07    1.0     0.82 ]
z_{C2}  [ 0.11  -0.09   -0.18    0.21    0.82    1.0  ]
```

对角线是自己和自己（永远为 1，训练时排除）。  
**正样本对**（同原图的两个增强版本）：$(A_1, A_2)$、$(B_1, B_2)$、$(C_1, C_2)$ —— 应该尽可能接近 1。  
**负样本对**（不同原图）：其余 30 个格子 —— 应该尽可能小。

**第四步：计算 NT-Xent 损失**

$$\ell(i,j) = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k \neq i}^{2N} \exp(\text{sim}(z_i, z_k) / \tau)}$$

以 $z_{A1}$ 为锚点，它的正样本是 $z_{A2}$，负样本是其余 4 个向量，$\tau=0.1$：

$$\ell(A_1, A_2) = -\log \frac{e^{0.87/0.1}}{e^{0.87/0.1} + e^{-0.12/0.1} + e^{0.05/0.1} + e^{-0.23/0.1} + e^{0.11/0.1}}$$

**直觉**：这就是一道选择题——从 5 个候选里找出"哪个是 A₁ 的正样本"。分子是正样本的得分，分母是所有候选的总得分。如果模型学得好（正样本相似度远高于负样本），分子占比大，$-\log$ 接近 0，损失小；如果模型学得差，5 个候选得分差不多，损失趋近 $\log 5 \approx 1.6$。

对批次里所有 $2N$ 个向量（每个都当一次锚点）计算这个损失并取平均，就是总 batch 损失。

**第五步：反向传播更新编码器**

损失对编码器参数求梯度。梯度的含义：对于 $z_{A1}$，
- **推近方向**：让编码器调整权重，使 A₁ 和 A₂ 的编码更接近（因为它们是同一张图的增强）
- **推远方向**：让编码器调整权重，使 A₁ 和 B₁、C₁ 等的编码更不相似（因为它们来自不同的图）

经过大量迭代，编码器学会了：**让图像的语义内容（而非随机增强的细节）决定向量位置**。同语义的图片聚在一起，不同语义的图片分开。

**为什么需要投影头？**  
编码器输出的特征（如 2048 维 ResNet 特征）直接做对比学习效果差。加一个 2 层 MLP 投影头（映射到 128 维）再做对比，下游微调时**扔掉投影头**，只用编码器的 2048 维特征。原因：对比任务的最优表示（对数据增强不变）和下游任务的最优表示（对语义敏感）之间有张力，投影头充当"缓冲层"，让编码器不被对比任务的过强约束所扭曲。

### 4.3 NT-Xent 损失公式解读

$$\ell(i,j) = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k \neq i}^{2N} \exp(\text{sim}(z_i, z_k) / \tau)}$$

**各符号含义：**
- $\text{sim}(u, v) = \frac{u \cdot v}{\|u\|\|v\|}$：余弦相似度，范围 $[-1, 1]$
- $\tau$（温度）：控制分布尖锐程度，通常取 0.07–0.1；越小分布越尖，困难负样本梯度越大
- $2N$：batch 中 $N$ 张图各有 2 个增强视图，共 $2N$ 个向量
- 分子：正样本对的相似度（希望大）
- 分母：对所有 $2N-1$ 个其他向量求和（全视为负样本）

负样本越多（batch 越大），分母越大，任务越难，被迫学到更有区分力的特征。这直接解释了为什么对比学习需要超大 batch（通常 4096+）。

### 4.4 局限

- 需要大量负样本，显存消耗巨大
- 数据增强策略必须人工设计，不同场景需要重新设计

## 五、冗余消除：Barlow Twins / VICReg

**代表**：Barlow Twins（Meta，2021）、VICReg（Meta，2022）

**核心思想**：不用负样本，直接约束表示向量的统计性质——让每个维度都有信息量，维度之间不重复。

### 5.1 一步训练的完整过程

同样以 **N=3** 张图、每张图 2 个增强视角为例。

**第一步：构造两组表示矩阵 Z 和 Z'**

```
原图 A → 增强A₁ → 编码器 → 投影头 → z_{A}  ∈ R^d
原图 B → 增强B₁ → 编码器 → 投影头 → z_{B}
原图 C → 增强C₁ → 编码器 → 投影头 → z_{C}

把这 N 个向量叠成矩阵：Z = [z_A; z_B; z_C]，形状 N×d

同样地，用另一套增强：
原图 A → 增强A₂ → ... → z'_{A}
原图 B → 增强B₂ → ... → z'_{B}
原图 C → 增强C₂ → ... → z'_{C}

Z' = [z'_A; z'_B; z'_C]，形状 N×d
```

**注意**：和对比学习不同，这里 $Z$ 和 $Z'$ 是两组并排的矩阵，每一行对应同一张图，不是放在一起的 $2N$ 个向量。三项损失分别在这两个矩阵上施加不同的约束。

---

**第二步：不变性损失——拉近同图的两个表示**

对矩阵里每一对同图的向量直接算 MSE：

$$s(Z, Z') = \frac{1}{N} \sum_{i=1}^{N} \| z_i - z'_i \|^2$$

假设 $d=3$（方便看），当前某张图 A 的两个表示为：

```
z_A  = [1.2,  -0.5,  0.8]
z'_A = [0.9,  -0.3,  1.1]

MSE = ((1.2-0.9)² + (-0.5+0.3)² + (0.8-1.1)²) / 3 = 0.073
```

这一项的梯度方向：**拉近** $z_A$ 和 $z'_A$，使同一张图的两个增强版本有相近的表示。

---

**第三步：方差损失——让每个维度不崩塌**

这是和对比学习最不同的地方。不是看同一张图的两个视角，而是**看这一批所有 N=3 张图在某个维度上的取值分布**。

假设编码器当前的 3 个向量是：

```
         维度0    维度1    维度2
z_A  = [  0.1,   0.2,    0.3 ]
z_B  = [  0.1,   0.2,    0.3 ]   ← 问题出现了：
z_C  = [  0.1,   0.2,    0.3 ]      所有图片在每个维度上输出一样的值！
```

这就是**维度崩塌（dimensional collapse）**：模型发现"对每张图都输出相同向量"也能让不变性损失为 0（因为 $z_i = z'_i$），但这样的表示没有任何区分度。

方差损失的作用：对每个维度 $j$，检查这 N 个样本的标准差。上例中维度 0 的标准差 = 0，惩罚极大。

```
方差损失要求：std([0.1, 0.1, 0.1]) > 1  → 失败，产生大惩罚

正常情况应该像：std([0.1, -0.8, 1.4]) > 1  → 通过
```

这强迫编码器：**不同图片在每个维度上的输出必须足够分散**——不能所有图片都映射到同一个点。

---

**第四步：协方差损失——让不同维度互不重叠**

假设维度 0 和维度 1 的关系如下（N=3 张图）：

```
图片   维度0   维度1
A     -1.0    -1.0
B      0.0     0.0
C      1.0     1.0
```

维度 0 和维度 1 完全相关（协方差 = 1）——两个维度表达了完全相同的信息，一个维度被浪费掉了。协方差损失惩罚这种情况，要求协方差矩阵的非对角元素接近 0。

直觉：**让每个维度"分工"，各自表达不同的信息**。维度 0 可能学会表达"颜色"，维度 1 学会表达"形状"，维度 2 学会表达"纹理"……维度之间的相关性越小，每个维度携带的信息越独立，整体表示空间的利用效率越高。

---

**三项损失的分工总结**

| 损失项 | 防止什么崩塌 | 作用范围 | 直觉类比 |
|--------|------------|---------|---------|
| **不变性** | 同图两视角差太远 | 行间（同一图的两个表示） | 拉紧绳子 |
| **方差** | 所有图映射到同一点 | 列内（同一维度，不同图） | 确保"每根维度弦"都有音量 |
| **协方差** | 不同维度表达相同信息 | 列间（不同维度之间） | 确保每根弦的音调不同 |

三项损失同时施加，编码器被迫找到一个平衡点：**同图不同视角的表示要接近（不变性），不同图的表示要分散（方差），每个维度要独立（协方差）**——这三个约束加在一起，就迫使编码器学出高质量、不冗余的语义表示。

---

**第五步：反向传播**

损失对编码器参数求梯度，梯度同时来自三项损失：
- 不变性梯度：拉近同图两表示
- 方差梯度：在每个维度上推开不同图片的表示
- 协方差梯度：减少维度间的相关性

**全程没有任何"这两张图应该不相似"的显式信号**——但方差项通过统计约束，间接保证了不同图片的表示不会全部堆在一起。

### 5.2 VICReg 三项损失公式

$$\mathcal{L} = \lambda \cdot s(Z, Z') + \mu \cdot [v(Z) + v(Z')] + \nu \cdot [c(Z) + c(Z')]$$

详细数学公式见**附录 A.1**。

**改进**：不需要负样本，更省内存；但仍需人工设计数据增强。

> **没有负样本如何防止不同图片的表示靠近？** 由**方差项**间接保证：方差项要求一批图片在每个维度上的取值必须足够分散（标准差 > 1），这隐式地要求不同图片的表示不能都堆在一起，但保证强度弱于显式负样本。实验证明这已经足够。

## 六、动量蒸馏：BYOL / DINO / DINOv2

**代表**：BYOL（DeepMind，2020）、DINO（Meta，2021）、DINOv2（Meta，2023）

**"动量"的含义**：不是物理动量，而是来自 MoCo 的命名。教师网络的权重通过 EMA 更新，每步只挪动极小一步（如 0.4%），表现出极强惯性——就像物理中质量很大的物体不容易被外力改变方向。"动量"= EMA 带来的惯性。

### 6.1 核心表征思想

对比学习用负样本推远，VICReg 用统计约束散开。动量蒸馏的思路完全不同：

> **"只要能一直准确预测另一个视角的表示，就说明你理解了这张图里的本质内容。"**

不需要负样本，不需要统计约束，只需要：**学生看到图片的一个视角，能预测出教师看到另一个视角时输出的表示向量**。

听起来似乎太简单了——如果学生直接输出一个全零向量，教师（它是学生的历史平均）也会输出全零向量，损失永远是 0，这不就崩塌了吗？

阻止崩塌的关键是**两个不对称设计**：
1. 学生有**预测头**（额外的 MLP 层），教师没有——两者结构不同，无法同时退化到同一个解
2. 教师比学生**更新慢得多**（EMA 每步 0.4% vs 梯度下降每步可能 10%+）——学生总是在追赶一个它还没追上的目标，"静止靶"才有可能被忽略，"移动靶"很难被全部忽略

第七节会详细分析这两个机制，这里先看训练流程。

### 6.2 一步训练的完整过程（以 BYOL 为例）

**第一步：构造两个视角**

```
原图 A
    ├─ 随机变换1（如裁剪左侧 + 翻转）→ 视角 A₁  → 给学生处理
    └─ 随机变换2（如裁剪右侧 + 调色）→ 视角 A₂  → 给教师处理
```

**第二步：两个网络分别前向传播**

```
视角 A₁ → 学生编码器 fθ → 特征 h₁ → 预测头 gφ → 预测输出 q
视角 A₂ → 教师编码器 fξ → 教师输出 z           （无预测头）

★ 关键：教师的输出 z 加了 stop-gradient，
         即 z 被当作"固定目标值"，梯度不会流向教师网络
```

**第三步：计算损失**

损失 = 学生预测 $q$ 与教师输出 $z$ 之间的**归一化 L2 距离**（等价于负余弦相似度）：

$$\mathcal{L} = \left\| \frac{q}{\|q\|} - \frac{z}{\|z\|} \right\|^2 = 2 - 2 \cdot \frac{q \cdot z}{\|q\| \cdot \|z\|}$$

损失越小，说明学生预测越准确，即学生从 A₁ 理解到的信息和教师从 A₂ 理解到的信息越一致。

训练时对所有批次图片求平均，再**反方向**：把 A₁ 给教师、A₂ 给学生，再算一次，两次损失取均值（对称化）。

**第四步：反向传播——只更新学生**

梯度从损失出发，反向传播路径：

```
损失 → 预测头 gφ（更新）→ 学生编码器 fθ（更新）→ 停止
                                                   ↑
                                  教师编码器 fξ 完全不参与，无任何梯度
```

**第五步：EMA 更新教师**

```
学生参数更新完成后，教师做 EMA：
fξ ← τ · fξ + (1-τ) · fθ    （τ ≈ 0.996）

含义：教师权重 = 学生近期历史的加权平均，每步只向学生靠近 0.4%
```

**训练的直觉**：学生不断尝试通过"自己这个视角的信息"预测"教师那个视角看到的结果"。预测越准，说明两个视角之间的不变信息（语义内容）被越好地捕获了。教师作为稳定的"移动靶"，随着训练进行慢慢进化，但始终领先学生一段距离。

---

**与对比学习、VICReg 的本质区别：**

| | 对比学习 | VICReg | 动量蒸馏（BYOL） |
|--|---------|--------|----------------|
| **防崩塌方式** | 负样本显式推远 | 统计约束（方差+协方差） | 教师-学生不对称 + EMA 移动靶 |
| **训练信号来源** | 批次内所有图片的相对关系 | 批次内的统计分布 | 仅来自同一张图的两个视角 |
| **batch size 要求** | 极大（4096+） | 较大（但小于对比学习） | 较小即可 |
| **理论理解程度** | 清晰（互信息最大化） | 清晰（信息论） | 部分是启发式的 |

### 6.3 DINO：把动量蒸馏用于 ViT，发现语义涌现

DINO（self-DIstillation with NO labels）在 BYOL 基础上做了几个关键调整，使其更适合 ViT：

- **Centering**：对教师输出减去一个批次均值，防止某些维度主导输出（BYOL 没有这一步）
- **Temperature sharpening**：教师输出用更小的温度参数，使分布更尖锐，给学生更明确的目标

**惊人的涌现现象**：用 DINO 训练出的 ViT，其注意力图（Attention Map）天然具有**语义分割能力**——即使从未接受过任何分割标注，模型的注意力权重已经能精确分离前景物体和背景。

这个发现暗示：仅靠自监督信号和足够大的 ViT，模型会自发学到高质量的语义理解。这是 DINO 在学术界引起轰动的原因。

### 6.4 DINOv2：把 DINO 做大

DINOv2（2023）在 DINO 基础上：
- 精心策划更大、更多样的训练集（LVD-142M，1.42 亿张精选图像，人工去除低质量和重复内容）
- 引入 **iBOT**（在线 token boosting）：将 DINO 的全图预测目标和 MAE 类似的**局部 patch 掩码预测**目标结合，同时训练全局语义理解和局部细节重建
- 模型规模扩展到 ViT-g（1B 参数）

结果：DINOv2 的特征质量在视觉表示基准上全面超越 CLIP/SigLIP，在**密集预测任务**（深度估计、语义分割）上尤其突出，部分 VLA 工作用 DINOv2 替代 SigLIP 作为视觉骨干。

## 七、教师-学生框架与 EMA

这是深度学习中一个非常重要的训练框架，贯穿 BYOL / DINO / I-JEPA，也在 WAM 的 DreamZero 中被使用。

### 7.1 原始含义：知识蒸馏（Knowledge Distillation，Hinton 2015）

**背景**：训练好了一个很大、很准的模型（教师），想压缩成小模型（学生）部署到手机上。

**做法**：用大模型的"软概率"输出（而非硬标签）训练小模型。软概率包含暗知识——"这张图和狗有点像"的信息在硬标签里完全丢失，而学生从这种软目标中学习更高效。

### 7.2 EMA（指数移动平均）

EMA = **Exponential Moving Average**，越近的历史权重越大，越老的权重越小（指数衰减）。

教师权重更新公式：

$$\xi \leftarrow \tau \cdot \xi + (1-\tau) \cdot \theta$$

其中 $\xi$ 是教师权重，$\theta$ 是学生权重，$\tau \approx 0.996$（每次只混入 0.4% 的学生权重）。

把公式展开可以看到，学生当前权重的贡献是 $(1-\tau)$，上一步是 $\tau(1-\tau)$，两步前是 $\tau^2(1-\tau)$……按指数衰减。$\tau=0.996$ 时，大约需要回溯 **250 步**之前的历史才衰减到可忽略——教师大致是学生过去 250 步的"历史平均"。

> **类比**：天气预报里的"近 7 天平均气温"是简单平均（每天权重相同），而 EMA 是"越近的天气越重要"的加权平均。

### 7.3 自监督中的教师-学生：BYOL / DINO

```
同一张图片 X
    ↓ 随机变换A（如裁剪左半）      ↓ 随机变换B（如裁剪右半）

学生网络 fθ（正常训练）           教师网络 fξ（慢速追随学生）
    ↓ 处理变换A                       ↓ 处理变换B
    ↓ + 预测头 g                      ↓
学生输出 q                        教师输出 z ← 作为目标

训练目标：让 q 尽量接近 z（最小化归一化 L2 距离）

更新方式：
  学生 θ：梯度下降（正常反向传播）
  教师 ξ：EMA 更新（不参与梯度，由学生历史平均得到）
```

**直觉**：教师是"昨天的你"。你（学生）每天努力学习，目标是让你的输出接近昨天的你对同一事物的判断。昨天的你是你近期历史的加权平均，两者之间永远存在一个追赶差距——这个差距就是防止崩塌的张力。

### 7.4 为什么教师不做梯度下降？

```
情况一：教师也做梯度下降
  → 两者同时朝损失最小值下降
  → 一起收敛到退化解（比如全输出零）→ 崩塌

情况二：教师只做 EMA
  → 教师是学生的"平滑历史"，不被当前梯度直接影响
  → 教师相对稳定，给学生提供有意义的"移动靶"
  → 学生无法找退化捷径 → 不崩塌
```

### 7.5 一步训练的完整执行顺序

```
① 前向传播（学生）
   变换A → 学生编码器 fθ → 特征 → 预测头 gφ → 输出 q

② 前向传播（教师）
   变换B → 教师编码器 fξ → 输出 z
   ★ stop-gradient：z 被当作常数，梯度不往教师这条路走

③ 计算损失
   Loss = ‖q/‖q‖ - z/‖z‖‖²（归一化 L2 距离）

④ 反向传播
   梯度只流经：预测头 gφ → 学生编码器 fθ
   教师编码器 fξ 完全不参与

⑤ 参数更新（两步独立执行）
   学生：θ ← θ - lr·∇θ        （梯度下降，幅度大）
   教师：ξ ← τ·ξ + (1-τ)·θ   （EMA，幅度极小）
```

**预测头的防崩塌作用**：学生有预测头，教师没有——这个结构不对称使崩塌更难发生。如果两者结构完全一样，优化压力更容易一起退化成崩塌解。

> **Q：教师是在向学生学习吗？** 是的，从权重角度看，教师的参数来自学生（EMA）。但"向谁学习"有两个方向同时存在：权重流向（学生 → 教师）和监督流向（教师 → 学生）。这形成一个**自举循环（Bootstrapping Loop）**——学生追赶教师，教师缓慢跟随学生，两者协同进化。

## 八、JEPA：预测表示而非像素

JEPA（Joint Embedding Predictive Architecture）是 Yann LeCun 提出的世界模型框架，在视觉 SSL 中的具体实现是 **I-JEPA**（图像）和 **V-JEPA**（视频）。

### 8.1 核心思路：为什么不预测像素，而要预测表示？

先用一个对比来建立直觉。假设你遮住了一张猫图的右耳区域，要求模型恢复这块区域：

**预测像素（MAE 的做法）**：  
模型需要输出每个像素的具体 RGB 值——右耳是什么颜色？毛发纹理是什么走向？光线从哪个方向打来？这些细节大部分是随机的、与"这是猫"这件事无关的信息。模型花了大量计算力去还原这些语义无关的细节。

**预测表示（JEPA 的做法）**：  
模型只需要输出"这块区域的语义表示应该是什么"——也就是目标编码器对这块区域的编码。这个编码已经是高维压缩过的，里面保留的是"这是猫耳朵，毛茸茸，三角形"这样的语义信息，而不是具体的像素颜色。预测难度低，语义价值高。

JEPA 的核心论点：**表示空间里的预测任务，天然过滤掉了不重要的低层细节，迫使模型只关注语义级别的内容**。

### 8.2 I-JEPA 的三个角色

I-JEPA 的训练涉及三个不同的组件，职责明确分开：

```
上下文编码器 fθ（学生，通过梯度下降更新）
  输入：图片的可见区域（部分 patch）
  职责：从可见内容里提取上下文语义

目标编码器 fξ（教师，EMA 慢速追随学生）
  输入：图片的完整内容（全部 patch）
  职责：提供"被遮盖区域应该有什么表示"作为监督目标
  重要：目标编码器看的是完整图片，不是残缺图片——
        它产生的目标表示包含了全局上下文信息

预测器 gφ（轻量 Transformer，和学生一起更新）
  输入：上下文表示 + 目标区域的位置编码
  职责：预测"目标区域的表示应该是什么"
```

目标编码器看**完整图片**这一点很关键：它能产生包含全局信息的目标表示（"这个位置是猫耳朵，并且这只猫正在草地上"），这比 MAE 的像素目标包含更丰富的语义信息。

### 8.3 掩码策略：块状遮盖而非随机稀疏

I-JEPA 和 MAE 的掩码方式不同：

```
MAE 的掩码：随机稀疏遮盖
  ████ ░░░░ ████ ████ ░░░░     （░ = 被遮盖，各处随机分布）
  ░░░░ ████ ░░░░ ████ ████
  → 每块遮盖面积小，周围有很多可见 patch 作为参考
  → 模型可以靠局部插值恢复，不需要理解全局语义

I-JEPA 的掩码：大型连续块状遮盖
  ████ ░░░░ ░░░░ ░░░░ ████     （大片连续区域被遮盖）
  ████ ░░░░ ░░░░ ░░░░ ████
  → 大面积遮盖，无法靠局部插值恢复
  → 模型必须理解全局场景，才能合理预测遮盖区域的表示
```

典型设置：对一张图，随机选 4 个大型目标块（每块约占图片面积的 15%–20%），保留一个小的上下文区域（约 15% 的 patch），其余遮盖。从极少量可见内容预测多个大面积遮盖块，任务难度远高于 MAE。

### 8.4 一步训练的完整过程

**第一步：划分上下文区域和目标区域**

```
一张 224×224 的图（被切成 196 个 patch，14×14 grid）

上下文区域（学生能看到的）：
  随机选约 30 个 patch（约 15% 的图片面积），可见

目标区域（需要预测的，共 4 块）：
  块1：位于右上角，12×12 = 144 个 patch
  块2：位于左下角，10×10 = 100 个 patch
  块3：位于中央，  8×8  = 64  个 patch
  块4：位于右下角，12×10 = 120 个 patch
  
  这些 patch 对学生不可见，只有位置信息（行列坐标）已知
```

**第二步：上下文编码器处理可见区域**

```
30 个可见 patch → 上下文编码器 fθ → 30 个上下文表示向量
                                      （每个向量 512 维）
```

**第三步：目标编码器处理完整图片**

```
196 个全部 patch → 目标编码器 fξ → 196 个表示向量

然后从 196 个向量中，
提取目标块1对应的 144 个向量 → s_y^(1)（这是块1的目标表示）
提取目标块2对应的 100 个向量 → s_y^(2)
提取目标块3对应的 64 个向量  → s_y^(3)
提取目标块4对应的 120 个向量 → s_y^(4)

★ stop-gradient：目标表示被当作固定目标值，梯度不流向目标编码器
```

**第四步：预测器预测目标表示**

预测器接收上下文表示 + 目标位置编码，对每个目标块分别预测：

```
[30个上下文向量] + [块1的位置编码] → 预测器 gφ → s_ŷ^(1)（预测的块1表示）
[30个上下文向量] + [块2的位置编码] → 预测器 gφ → s_ŷ^(2)
[30个上下文向量] + [块3的位置编码] → 预测器 gφ → s_ŷ^(3)
[30个上下文向量] + [块4的位置编码] → 预测器 gφ → s_ŷ^(4)
```

**第五步：计算损失，反向传播**

$$\mathcal{L} = \sum_{k=1}^{4} \left\| s_{\hat{y}}^{(k)} - \text{stop\_grad}(s_y^{(k)}) \right\|^2$$

损失只反传到**预测器**和**上下文编码器**，目标编码器不参与梯度。

**第六步：EMA 更新目标编码器**

```
fξ ← τ · fξ + (1-τ) · fθ    （τ ≈ 0.996）
```

**训练的直觉**：上下文编码器从极少量可见 patch 里提取语义，预测器用这些语义加上位置信息，尝试重建"如果能看到那块区域，应该有什么语义表示"。这迫使编码器建立对整个场景的全局语义理解，而不只是记住局部纹理。

### 8.5 I-JEPA vs BYOL / DINO 的关键区别

| | BYOL / DINO | I-JEPA |
|--|-------------|--------|
| **教师看什么** | 同一图片的另一种增强视角（残缺视图） | 图片的**完整内容** |
| **学生看什么** | 同一图片的一种增强视角 | 图片的**部分 patch**（可见区域） |
| **预测目标** | 教师对"整图另一视角"的全局表示 | 教师对"被遮盖块"的局部表示 |
| **位置信息** | 无需（两个视角不区分位置） | 关键（预测器需要知道"预测哪个位置"） |
| **需要数据增强** | 是（关键，每种模态需单独设计） | 否（掩码本身是任务，模态无关） |

**I-JEPA 最重要的改进**：任务设计本身（哪些区域可见、哪些需要预测）不依赖任何人工设计的数据增强，只依赖图像内部的空间结构——因此方法可以直接迁移到视频（时空 patch）、音频（时间段）等其他模态，无需为每种模态设计增强策略。

### 8.6 V-JEPA：扩展到视频

V-JEPA（2024）把 I-JEPA 的思路扩展到视频：遮盖**时空 patch 块**（某几帧里的某个区域，时间上连续），预测被遮盖的时空区域在表示空间里的表示。

```
视频 = (T帧, H×W像素) → 被切成 (T × H/p × W/p) 个时空 patch

目标块：比如第 5–8 帧里的右上角区域
上下文：第 1–4 帧的全部 patch + 第 5–8 帧其余区域的 patch

预测任务：给定能看到的时序上下文，预测第 5–8 帧右上角会是什么表示
→ 模型必须理解物体的运动规律才能合理预测
```

### 8.7 JEPA 与像素空间世界模型的关系

JEPA 是一种**潜空间世界模型**（在表示空间里预测），与 DreamZero/LingBot-VA 的**像素空间世界模型**（预测真实视频帧）形成对比。两者的设计哲学详见 [世界模型专题 第二节](世界模型.md)。

## 九、CLIP：跨模态对比学习

**CLIP（Contrastive Language-Image Pre-Training，OpenAI，2021）** 是现代 VLM 和 VLA 中最广泛使用的视觉编码器之一，理解它是理解 VLA 工作方式的关键。

### 9.1 模型结构：两个独立的编码器

CLIP 有两个编码器，分别处理两种模态：

```
图片编码器（Image Encoder）
  结构：ViT（Vision Transformer）或 ResNet
  输入：一张图片（如 224×224 像素）
  输出：一个向量（如 512 维）

文字编码器（Text Encoder）
  结构：Transformer（类似 GPT-2）
  输入：一段文字（如 "a photo of a cat"）
  输出：一个向量（同样 512 维）
```

关键：两个编码器的输出向量**维度相同**，最终都被 L2 归一化到单位球面上——这样两个向量的点积就等于余弦相似度，范围在 $[-1, 1]$。

**两个编码器之前完全独立，什么都不共享**，它们通过训练目标被"拉到同一个空间"里。

### 9.2 训练数据：互联网上的图文配对

CLIP 的训练数据是从互联网收集的 **4 亿个（图片, 文字描述）配对**，例如：

```
图片：一只橘猫坐在草地上   ←→   文字："a ginger cat sitting on the grass"
图片：一个红色咖啡杯        ←→   文字："red coffee mug on a white table"
图片：机器人手臂操作零件    ←→   文字："robotic arm assembly in factory"
```

这些数据**不需要人工标注**，直接来自网页上图片自带的 alt 文本、图注、周围段落等——数据量极大但噪声也很大（图片和文字描述往往只是"大致相关"而非精确对应）。

### 9.3 一步训练的完整过程

以 **N=4** 个图文配对为例。

**第一步：分别编码图片和文字**

```
批次数据（4个配对）：
  (图1: 橘猫,   文1: "a ginger cat")
  (图2: 红杯子, 文2: "red coffee mug")
  (图3: 机器人, 文3: "robotic arm")
  (图4: 海滩,   文4: "beach sunset")

图片编码器分别处理 4 张图：
  图1 → v₁ = [0.2, -0.5, 0.8, ...]（512维向量，已L2归一化）
  图2 → v₂ = [-0.1, 0.9, 0.3, ...]
  图3 → v₃ = [0.7, 0.1, -0.4, ...]
  图4 → v₄ = [-0.3, -0.2, 0.6, ...]

文字编码器分别处理 4 段文字：
  文1 → t₁ = [0.18, -0.52, 0.76, ...]（同维度，已L2归一化）
  文2 → t₂ = [-0.08, 0.88, 0.31, ...]
  文3 → t₃ = [0.69, 0.09, -0.42, ...]
  文4 → t₄ = [-0.28, -0.19, 0.62, ...]
```

**第二步：计算 N×N 相似度矩阵**

把 4 个图片向量和 4 个文字向量两两做点积（= 余弦相似度），得到 4×4 矩阵：

```
             文1      文2      文3      文4
图1        [ 0.95    0.07    -0.12    0.03  ]  ← 图1 和文1 是配对，相似度高
图2        [ 0.08    0.93     0.05   -0.09  ]  ← 图2 和文2 是配对
图3        [-0.10    0.04     0.91    0.08  ]  ← 图3 和文3 是配对
图4        [ 0.02   -0.11    0.06    0.94  ]  ← 图4 和文4 是配对
```

训练目标：**对角线（配对）的相似度尽量接近 1，非对角线（不配对）的尽量接近 0（或负数）**。

训练初期这个矩阵是随机的，对角线没有特别高——损失推动两个编码器调整权重，使配对的图文向量越来越接近。

**第三步：计算对称对比损失**

CLIP 用对称的方式计算损失——把"给图片找对应文字"和"给文字找对应图片"都算一遍：

**图片方向**（以图1为锚点，在4段文字里找配对文字）：

$$\ell_{\text{图→文}}(1) = -\log \frac{e^{v_1 \cdot t_1 / \tau}}{\sum_{j=1}^{4} e^{v_1 \cdot t_j / \tau}}$$

用上面的数字，$\tau=0.07$：

$$= -\log \frac{e^{0.95/0.07}}{e^{0.95/0.07} + e^{0.07/0.07} + e^{-0.12/0.07} + e^{0.03/0.07}}$$

分子极大（正确配对相似度高），分母中其他项极小，损失接近 0——模型知道图1配文1。

**文字方向**（以文1为锚点，在4张图里找配对图片）：

$$\ell_{\text{文→图}}(1) = -\log \frac{e^{t_1 \cdot v_1 / \tau}}{\sum_{i=1}^{4} e^{t_1 \cdot v_i / \tau}}$$

对所有 $N$ 个配对的两个方向求平均，得总损失：

$$\mathcal{L}_{\text{CLIP}} = \frac{1}{2N}\left[\sum_{i=1}^{N}\ell_{\text{图→文}}(i) + \sum_{i=1}^{N}\ell_{\text{文→图}}(i)\right]$$

**第四步：反向传播，同时更新两个编码器**

梯度同时流向图片编码器和文字编码器，两者**一起**被更新。训练的效果：

- 图片编码器学会把"橘猫的图片"编码到和"a ginger cat"的文字编码相近的向量空间位置
- 文字编码器学会把"a ginger cat"编码到和"橘猫图片"相近的空间位置
- 经过 4 亿个图文对的训练，两个编码器的向量空间被对齐：**相同语义的图片和文字会落在空间的同一区域**

### 9.4 训练好之后能做什么

**零样本分类**（CLIP 最著名的能力）：不需要任何额外训练，直接做分类。

```
新图片：一只金毛犬
候选文字："a photo of a dog" / "a photo of a cat" / "a photo of a bird"

计算图片向量与三段文字向量的余弦相似度：
  "dog"：0.91  ← 最高
  "cat"：0.12
  "bird"：-0.05

→ 判断为"dog"，正确！
```

这之所以能工作，是因为 CLIP 的文字编码器已经理解了"dog"和"cat"的语义差异，图片编码器已经学会把图片特征对齐到这个语义空间。

**在 VLA 中的作用**：

| CLIP 学到的能力 | 在 VLA 中如何用 |
|--------------|--------------|
| 图片向量与文字向量在同一空间 | 视觉 token 和语言 token 可以直接 Cross-Attention，不需要额外对齐模块 |
| 4 亿图文对的语义理解 | VLA 视觉编码器直接继承 CLIP 权重，零成本获得颜色/物体/场景理解 |
| 对"把 X 放到 Y 上"等语言描述的视觉理解 | 让 VLA 能理解语言指令与视觉场景的关系 |

### 9.5 SigLIP：CLIP 的改进版

CLIP 的 Softmax 对比损失有一个问题：对**单个图文对**来说，计算损失时必须把整个 batch 里所有 N 个文字（或图片）都考虑进来作为分母——batch 越大，计算越精确，但显存消耗也越大。

**SigLIP（Sigmoid Loss for Language-Image Pre-Training，Google，2023）** 做了一个简单但有效的改变：把 Softmax 换成 Sigmoid。

```
CLIP（Softmax）：
  对每张图片，在 N 段文字里做一个 N 选 1 的分类
  → 必须同时看所有 N 段文字，batch 内强耦合

SigLIP（Sigmoid）：
  对每个（图片, 文字）对，独立判断"这对是不是配对"（二分类）
  → 每对独立计算，batch 内解耦
```

SigLIP 的损失：

$$\mathcal{L}_{\text{SigLIP}} = -\frac{1}{N^2}\sum_{i,j} \log \sigma\!\left(y_{ij} \cdot (v_i \cdot t_j / \tau - b)\right)$$

其中 $y_{ij} = +1$ 表示配对，$y_{ij} = -1$ 表示不配对，$b$ 是偏置项，$\sigma$ 是 Sigmoid 函数。

优点：对大 Batch 更友好（不需要 Softmax 归一化），在下游任务上表现更好。目前 OpenVLA、π₀ 等主流 VLA 均使用 SigLIP 而非原始 CLIP。

### 9.6 CLIP 的局限

- 对**细粒度空间关系**（"物体 A 在物体 B 的左边"）理解薄弱——互联网图文对描述的是语义，很少精确描述空间关系
- 对**接触力学、物体质量**等物理属性没有理解——互联网图片缺乏物理标注
- 遇到训练分布外的物体（工业零件、医学影像）表现退化

## 十、各方案横向对比

| 方法 | 类型 | 需要负样本 | 需要数据增强 | 预测目标 | 代表 |
|------|------|-----------|------------|---------|------|
| **对比学习** | 判别式 | 是（大量） | 是（关键） | 表示相似度 | SimCLR、MoCo |
| **冗余消除** | 判别式 | 否 | 是 | 表示相似度 + 统计约束 | Barlow Twins、VICReg |
| **动量蒸馏** | 判别式 | 否 | 是 | 教师输出的表示 | BYOL、DINO、DINOv2 |
| **MAE** | 生成式 | 否 | 否（掩码是任务） | 像素值 | MAE、VideoMAE |
| **JEPA** | 生成式（表示空间） | 否 | 否 | 表示向量 | I-JEPA、V-JEPA |
| **CLIP** | 跨模态对比 | 是（图文配对） | 否 | 跨模态对齐 | CLIP、SigLIP |

**关键趋势**：
1. 从"需要精心设计数据增强"走向"用数据本身的结构（掩码/多模态配对）做监督"
2. 从"仅图像"走向"跨模态对齐（图+文）"——CLIP 和 SigLIP 代表了这个方向的胜利
3. DINO/DINOv2 的语义涌现和 MAE 的细粒度重建互补，在很多实际系统中两者合并使用

## 十一、与 LLM / VLA / WAM 的连接

自监督表征学习是这些系统的基础设施层，了解它有助于理解"为什么 VLA 能开箱即用"。

**LLM（语言模型）**：GPT/BERT 的预训练本质上就是自监督学习（预测下一 token / 掩码填充），只不过对文本来说这太自然，反而不常被叫"SSL"。

**VLM / VLA 视觉编码器**：
- OpenVLA / π₀ 使用 **SigLIP**（CLIP 的改进版）作为视觉编码器
- 部分工作使用 **DINOv2** 作为视觉骨干
- 这些编码器输出的视觉 token 能直接和语言 token 做 Cross-Attention，正是因为 CLIP 的图文对齐训练

**WAM 的 IDM**：LingBot-VA 用逆动力学模型从视频帧间差异推导动作，这个"视频帧的表示"依赖预训练良好的视觉编码器——如果视觉表示很差，IDM 就无法从帧间差异里提取有意义的动作信息。

**JEPA 与 WAM 的关系**：JEPA 是"在表示空间预测未来"，WAM 是"在像素空间联合预测未来帧和动作"——两者都属于世界模型的思路，但 WAM 选择像素空间是为了能直接输出可执行的控制信号（见 [世界模型专题 第二节](世界模型.md)）。

## 十二、前沿发展（2025-2026）

### 12.1 SSL 进入多模态统一时代

- **ImageBind**（Meta，2023）：把图像、文字、音频、深度图、IMU 传感器等 6 种模态都对齐到同一个表示空间，只用图像作为桥梁（其他模态分别与图像做对比学习）。首次展示了多模态统一表示的可行性。
- **VideoChat2 / InternVideo2**（2024）：把视频理解的 SSL 预训练扩展到"视频-音频-文字"三模态对齐，为视频理解模型提供更丰富的表示

### 12.2 具身智能专用 SSL

- **R3M**（Meta，2022）：用人类视频（Ego4D 数据集）做时序对比学习，专门为机器人操作任务设计的表示预训练，提取的特征在机器人任务上远好于 CLIP
- **SPA**（Spatial Awareness Pretraining，Microsoft Magma，2025）：在空间感知任务（物体位置、相对关系）上预训练视觉编码器，为 2D（Web）和 3D（机器人）操作提供统一的空间理解表示

### 12.3 SSL 与生成式模型的界限模糊

扩散模型和 JEPA 的交叉让"SSL"和"生成模型"的边界越来越模糊：
- **Diffusion-based representation learning**：用扩散模型的中间特征作为下游任务的表示
- **Generative SSL**：把生成质量作为 SSL 质量的衡量标准

两条路线正在向同一个目标汇聚：学到既能生成、又能理解的统一表示。

## 📎 附录

### A.1 VICReg 和 Barlow Twins 详细公式

> 关联正文：**第五节 → 冗余消除**

**VICReg 三项损失的具体公式**（$N$ = batch size，$d$ = 表示维度）：

**① 不变性损失**（直接 MSE）：
$$s(Z, Z') = \frac{1}{N} \sum_{i=1}^{N} \| z_i - z'_i \|_2^2$$

**② 方差损失**（逐维度检查标准差）：
$$v(Z) = \frac{1}{d} \sum_{j=1}^{d} \max\!\left(0,\ \gamma - \sqrt{\text{Var}(z^j) + \epsilon}\right), \quad \gamma=1$$

**③ 协方差损失**（让非对角元素为零）：
$$c(Z) = \frac{1}{d} \sum_{i \neq j} \left[C(Z)\right]_{ij}^2, \quad C(Z) = \frac{1}{N-1} \sum_{i=1}^N (z_i - \bar{z})(z_i - \bar{z})^\top$$

**Barlow Twins**：直接计算 $Z_1$ 和 $Z_2$ 之间的互相关矩阵，要求对角线为 1（正样本对相关）、非对角线为 0（维度去相关）：

$$\mathcal{L}_{BT} = \underbrace{\sum_{i} (1 - \mathcal{C}_{ii})^2}_{\text{不变性}} + \lambda \underbrace{\sum_{i \neq j} \mathcal{C}_{ij}^2}_{\text{冗余消除}}$$

两者区别：VICReg 对 $Z_1$、$Z_2$ 分别施加统计约束；Barlow Twins 直接约束两者之间的相关结构，更接近信息论视角。

### A.2 温度参数 τ 的作用

> 关联正文：**第四节 → 对比学习 NT-Xent 损失**

NT-Xent 损失中的 $\tau$（通常取 0.07–0.5）控制相似度分布的"尖锐程度"：

- $\tau$ 小（如 0.07）：softmax 分布非常尖锐，模型对"哪个是正样本"非常确定。训练信号强但容易过拟合，对数据增强策略敏感。
- $\tau$ 大（如 0.5）：softmax 分布平滑，训练信号弱但更稳定。

SimCLR 发现 $\tau=0.1$ 是较优的选择。$\tau$ 本质上控制了损失函数对"困难负样本"（与正样本相似的负样本）的关注程度——$\tau$ 越小，困难负样本对梯度的贡献越大。
