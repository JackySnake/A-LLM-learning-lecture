# VLM 与 LLM：关系、张力与开放难点

*触发来源：学习 Week 8 多模态架构设计时，对"VLM 与 LLM 之间的微妙关系"产生系统性疑问——为什么顶级厂商仍主攻 LLM？多模态真的让模型更强了吗？为什么架构、建模、后训练、长度处理至今没有定论？（2026-06-21）*

> **前置知识**：Week 8（多模态架构形态、训练管线、M-RoPE、长序列处理）。本文不重复 Week 8 的"怎么做"（架构与技术），而是回答更上一层的"为什么这么难、哪些还没有答案"。可视为 Week 8 的**问题地图（problem map）**。
>
> **本文定位**：这是一份**问题导向**的文档，不是结论手册。很多问题至今是开放的——本文的价值在于把它们摆清楚、给出当前最好的认知与证据、标明哪些还悬而未决，为后续深入留接口。文末第九节列出了可继续深挖的方向。

---

## 一、破题：VLM 不是 LLM 的超集

学习多模态时，最容易先入为主的一个直觉是：

> "VLM = LLM + 眼睛。既然能看又能读，VLM 应该是 LLM 的超集，能力只会更强。"

**这个直觉是错的，而且错得很关键。** 你提到的所有困惑，根子都在这里。我们先把这个命题彻底掰开，它会统领后面所有章节。

### 1.1 为什么"多模态智能 > 语言智能"站不住

把语言和视觉放在"信息论"的天平上看，它们是两种性质完全不同的信号：

| 维度 | 语言（文本） | 视觉（图像/视频） |
| :--- | :--- | :--- |
| **信息密度** | **极高**：每个 token 是人类压缩了千百年的抽象符号 | **极低**：相邻像素高度冗余，绝大部分比特无语义 |
| **抽象层级** | 已经是高层抽象（"猫"这个词本身就是概念） | 原始感知信号，概念需要从像素里**重新提取** |
| **携带的知识** | 承载人类显式知识、逻辑、推理链 | 承载物理世界的外观、空间、动态，但极少显式推理 |
| **是否为推理载体** | **是**：人类的推理几乎都以语言/符号进行 | 否：视觉本身不"推理"，它提供素材 |

关键洞察：**语言是人类智能数千年压缩沉淀的产物，它本身就是一种"近乎最优编码"的高密度知识载体。** 而视觉是原始、冗余、低密度的感知流。

于是"多模态一定更强"的逻辑链就断了：

- 高级**推理**能力主要寄生在语言这个高密度符号系统上。给模型灌入大量低密度的视觉信号，**不会自动提升推理**，反而可能稀释、挤占模型在语言上的容量（这就是第二节的"多模态税"）。
- 视觉能补的是**感知与具身**（看得见、定位、理解空间动态），这是语言天然缺失的维度——所以多模态在"感知类任务"上确实强于纯语言。但"感知强" ≠ "智能强"。

> **一句话**：VLM 在**感知维度**上扩展了 LLM，但在**推理维度**上并不天然超越 LLM，甚至要花力气才能"不退步"。VLM 与 LLM 是**交集而非包含**关系。

### 1.2 为什么顶级 LLM 厂商仍重金主攻 LLM

这就解释了你观察到的现象。原因可以收敛成三条：

1. **推理的天花板由语言决定**。当前能力竞赛的主战场是 reasoning（数学、代码、agent）。这些能力的载体是语言/符号链，提升它们最有效的杠杆仍是更强的语言模型 + RL，而不是加视觉。
2. **多模态有"税"没"红利"（在推理上）**。引入多模态训练，纯文本能力常常**下降**（实证见第二节）。对一个主打推理的旗舰模型，这是负收益。
3. **数据与评测的不对称**。高质量文本数据 + 可验证奖励（数学/代码对错可判定）让 LLM 的训练飞轮转得很快；多模态的数据质量、评测可靠性都差一截（见第七节评测危机）。

所以业界的真实形态往往是：**先把语言基座做到极强，再"小心翼翼"地接上视觉，并想办法别把语言能力搞坏。** 多模态是能力的**外延**，而非智能的**内核**。

## 二、能力会因多模态而退化吗？——"多模态税"（Multimodal Tax）

> **对应你的问题**："引入多模态并没有使模型更强，甚至在原来 LLM 做得好的工作上性能受损。"

这不是错觉，是被反复实证的现象，业界半正式地称之为 **multimodal tax（多模态税）** 或 **modality interference（模态干扰）**。

### 2.1 实证证据（这是你这个观察的硬支撑）

把一个强 LLM 基座接上视觉、做完多模态训练后，回头测它的**纯文本**能力，普遍掉点：

- **VILA-1.5 40B**：纯文本基准从 54.8 → 47.9，掉 **6.9 分**；
- **LLaVA-OneVision 72B**：掉约 **6.3 分**；
- **InternVL-2-Llama3-76B**：掉约 **6.9 分**；
- 更尖锐的是：**多模态 RL 训练会损害纯文本的数学推理**（GSM8K、MATH-500 上掉点）——也就是说连后训练阶段的多模态对齐都会"反噬"语言推理。

（来源：NVLM 论文 2409.11402 的对比分析；以及多篇关于多模态 RL 损害文本推理的工作。）

### 2.2 为什么会发生（机制）

1. **参数共享的"跷跷板"**。视觉和语言共用同一套 Transformer 权重。当海量视觉 token 参与训练，梯度会把权重往"适配视觉"的方向推，挤占原本服务语言推理的表征容量。这本质是**容量竞争**。
2. **表征干扰**。视觉 token 被投影进语言的隐空间后，与文本 token 共享注意力。低密度的视觉信号会"稀释"注意力分布（第六节会再展开）。
3. **灾难性遗忘**（连 [Week 5 灾难性遗忘 & 持续学习]）。多模态训练是一次大规模的领域迁移，纯文本能力作为"旧任务"被部分遗忘——这与 Week 5 讨论的持续学习困境是同一个数学问题。

### 2.3 缓解手段（以及它揭示的深层事实）

- **回灌纯文本数据**：训练时混入高质量 text-only 指令/推理数据。NVLM 这么做后，72B 模型纯文本平均分从 79.8 **反超** 到 84.1——说明税是可以对冲甚至转正的，但要刻意设计。
- **选择性蒸馏**：用冻结的纯语言 teacher 把语言能力"蒸"回 VLM（如 LinguDistill 一类工作）。
- **早期低比例均衡融合**：从预训练第一天就以**固定低比例**持续混入视觉数据，而非后期集中灌（Week 8 提到的 Kimi K2.5 工程洞察就是这个思路）。

> **开放问题**：能不能做到"只增不减"——加了视觉，语言一分不掉甚至更强？目前只能靠数据配比/蒸馏**对冲**，没有从根上消除税的优雅方案。这是 VLM 训练最现实的痛点之一。

## 三、架构为什么没有统一定论

> **对应你的问题**："VLM 架构没有像 LLM 那样的统一定论；闭源被认为已普遍采用原生统一架构（但实证不强）；LLaVA 式非原生架构仍有团队在做且有优势。"

你的判断很准。这里有两个层次的"不统一"。

### 3.1 第一层：原生（Native / Early Fusion）vs 模块化（Modular / LLaVA 式 Late Fusion）

Week 8 Part 1 已经讲了二者的技术区别。这里补的是**为什么这场争论至今没分出胜负**——因为两条路各有不可替代的优势：

| | 模块化（LLaVA 式：ViT + Projector + LLM） | 原生（Early Fusion：从预训练就联合） |
| :--- | :--- | :--- |
| **复用强 LLM** | ✅ 直接接最强的现成 LLM 基座 | ❌ 通常要从头/早期联合训练 |
| **训练成本** | ✅ 便宜，对齐阶段只训 Projector 即可起步 | ❌ 昂贵，要海量图文交错数据 |
| **换组件灵活性** | ✅ 换 encoder、换 LLM 都很方便 | ❌ 牵一发动全身 |
| **跨模态融合深度** | ❌ 有信息瓶颈（Projector 是窄口） | ✅ 全层深度融合，理论上限更高 |
| **多模态税** | 相对可控（语言基座可冻结/少动） | 更难控（联合训练更易干扰语言） |

**结论**：模块化是**工程务实**路线（便宜、灵活、能蹭最强 LLM），原生是**能力上限**路线（融合更深，但贵且难）。学术界、创业公司大量用模块化（因为便宜灵活）；大厂旗舰倾向原生（因为追上限）。两者不是谁淘汰谁，而是**不同约束下的不同最优解**。

### 3.2 第二层："原生"内部也不统一

更微妙的是：你说的"闭源已普遍原生"这个判断本身**证据偏弱**——它更多是社区的**信念**而非定论（闭源不公开架构）。而且即便都叫"原生"，内部仍分裂成 Week 8 讲的几种子范式：

- **连续 token 早融合**（Gemini、GPT-4o、Qwen-Omni）——理解强；
- **离散 token 统一自回归**（Chameleon、Emu3.5）——能统一生成但有量化损失；
- **混合式**（Janus-Pro 的解耦双编码器、Transfusion 的扩散生成）——试图两全。

这些"原生"之间的差异，**比原生 vs 模块化的差异还大**。所以严格说：**VLM 架构连"原生派内部"都没统一。**

### 3.3 为什么 LLM 收敛了而 VLM 没有

LLM 架构早早收敛到 "Decoder-only + NTP"，因为：① 文本只有一种模态、一种无损 tokenizer（BPE）；② 一个训练目标（next token）通吃理解与生成。

VLM 不具备这两个前提（见第四节）：视觉**没有公认的无损 tokenizer**，理解与生成**没有统一的目标**。前提不成熟，架构自然无法收敛。**架构的不统一，是建模问题不统一的"果"。**

> **开放问题**：是否存在"the architecture"（一统天下的 VLM 架构），还是说架构本就该**任务依赖**（理解任务、生成任务、具身任务各有最优）？目前没有答案。

## 四、建模范式：NTP 迁移到多模态的根本困难

> **对应你的问题**：这是你问得最深的一组——"LLM 的 NTP 迁移到 VLM，非语言部分用 NTP 不能直接用；若只生成语言，非语言部分就没有 NTP 训练信号；若要任意模态都能输出，就进入多模态理解与生成如何统一的深水区。"

你已经摸到了 VLM 建模的**核心裂缝**。我们把它系统化。

### 4.1 文本 NTP 为什么那么"好用"——以及它的两个隐藏前提

NTP（Next-token Prediction）在文本上几乎是完美的自监督目标，但它依赖两个**默默成立**的前提：

1. **无损离散化**：BPE 把文本切成离散 token，且**可完美还原**（token 序列 ↔ 原文一一对应，无信息损失）。
2. **理解与生成同构**：预测下一个词，既训练了"理解上文"，又训练了"生成下文"。**一个目标，两件事一起做完。**

视觉把这两个前提**全破坏了**——这就是麻烦的根源。

### 4.2 破裂点一：视觉没有"无损 tokenizer"（这是最被低估的难点）

文本 BPE 是**无损、固定、免训练**的；视觉的 tokenizer 却是**有损、需要学习**的组件：

- 走**离散路线**（VQ-VAE/VQ-GAN）：把图像 patch 映射到有限码本 ID。码本容量有限 → **量化损失**，高频细节被抹掉。这是结构性短板（Week 8 范式 B 已述）。
- 走**连续路线**（ViT 特征）：保真度高，但视觉特征**不在离散词表里**，无法直接用 NTP "吐出"。

换句话说：**视觉信号要么能进 NTP 框架但有损（离散），要么保真但出不来（连续）。** 文本里那个"无损 + 可生成"的甜蜜点，视觉根本不存在。这是 VLM 建模无法照搬 LLM 的第一性原因。

### 4.3 破裂点二：理解与生成在视觉上"撕裂"了

这正是你说的那个分叉：

- **如果只要模型输出语言**（绝大多数 VLM：看图说话、VQA）：那么视觉部分**只做输入、不做输出**，它**没有 NTP 监督信号**——视觉表征只能靠"预测下一个文字"间接被训练（梯度从语言 loss 回传到视觉投影层）。这导致视觉理解的训练信号**很弱、很间接**，是 VLM "看不清细节"的根源之一（连第七节感知天花板）。
- **如果要模型也能输出视觉**（任意模态输出 / any-to-any）：那就必须给视觉一个生成目标。于是进入你说的深水区——**理解与生成如何统一**。

而这个统一，碰到一个被反复验证的硬墙：**理解-生成优化困境（understanding-generation optimization dilemma）**。

### 4.4 理解-生成优化困境

2025–2026 的统一多模态研究（综述见 arXiv 2505.02567《Unified Multimodal Understanding and Generation Models》；专门分析见《Understanding vs. Generation: Navigating Optimization Dilemma》）给出的核心结论是：

> **在同一个模型里，增强生成能力常常以损害理解为代价，反之亦然——二者存在竞争性张力。**

为什么冲突？**因为理解和生成需要的视觉特征不一样**（这正是 Week 8 范式 C-1 Janus 解耦双编码器验证的洞察）：

- **理解**需要**全局语义抽象**（"这是一只在跑的猫"）；
- **生成**需要**像素级局部细节**（每根毛、光影、纹理）。

强行让一套表征同时服务两个方向，目标互相拉扯。业界的应对：

- **解耦编码器**（Janus：理解用 SigLIP 连续特征，生成用 VQ 离散）——缓解冲突但架构不再"统一"；
- **双目标共存**（Transfusion：文本走 NSP，图像走 Diffusion loss，同一个 Transformer 内并存两种 loss 与两种注意力掩码）——目前工业级最成功的方案，但本质是"在一个壳里塞两个目标"，而非真正用一个目标统一。

### 4.5 反向问题：生成能帮助理解吗？（analysis by synthesis）

有一个很深的假设：**"能生成才算真懂"**（要画出一只猫，你必须真的理解猫长什么样）。研究证据是**双向的**：

- 支持：辅助生成内在视觉表示（深度、分割、像素）可改善细粒度感知、减少幻觉、增强空间理解；有工作把生成重构为"生成→理解→再生成"的多步过程来反哺理解。
- 反对：前述优化困境表明，naive 地加生成目标反而拖累理解。

> **开放问题**：VLM 至今**没有一个像 NTP 之于文本那样"一个目标通吃理解与生成"的统一训练目标**。这是比"架构不统一"更底层的不统一。统一的建模目标，是这个领域最重要的圣杯之一。

## 五、后训练：VLM 和 LLM 的对齐要求一样吗？

> **对应你的问题**："VLM 和 LLM 的后训练要求一样吗？多模态能力是否会溢出到语言上？这是否要在后训练阶段考虑？"

答案是：**不一样，而且多模态在后训练阶段引入了 LLM 没有的新麻烦，同时你担心的"溢出/干扰"是真实存在的。**

### 5.1 多模态后训练独有的难题

1. **视觉幻觉（object hallucination）**：模型"看见"图里没有的东西，或无视图像、凭语言先验编答案。这是 VLM 特有的对齐目标，纯 LLM 没有（连第七节评测危机——很多 VLM 根本没在看图）。
2. **视觉 grounding / 指代对齐**：回答要能**落到图像具体区域**（"左下角那个红色按钮"），需要专门的 grounding 数据与奖励。
3. **奖励建模更难**：RLHF 的奖励模型在多模态下**也得看图**才能判断"答得对不对、看得准不准"。一个看不准图的奖励模型会引导出 reward hacking（生成听起来对、其实没对上图的答案）。
4. **多模态安全**：图像越狱（把有害指令藏进图片里绕过文本安全对齐）是 LLM 完全没有的攻击面。

### 5.2 你担心的"溢出/干扰"是真的

这其实是第二节"多模态税"在**后训练阶段**的体现，而且更尖锐：

- **多模态 → 语言（负向溢出）**：前面说过，**多模态 RL 会损害纯文本数学推理**（GSM8K/MATH 掉点）。所以后训练**必须**把"保住语言能力"当成一个显式约束（混入纯文本 RL/SFT 数据）。
- **语言 → 多模态（正向溢出）**：强语言推理能力可以**迁移**到视觉推理（"先理解图、再用语言链条推理"）。这是好事，也是为什么"强 LLM 基座 + 接视觉"路线有效。

> **结论**：VLM 后训练 ≠ LLM 后训练 + 视觉数据。它需要**额外的对齐目标**（抗幻觉、grounding、视觉安全）+ **显式的语言能力保护**（防多模态税）+ **能看图的奖励模型**。"多模态能力是否溢出到语言"——不仅会，而且必须在后训练阶段主动管理。

> **开放问题**：是否存在统一的"混合后训练"配方，能同时优化语言推理、视觉理解、视觉生成而不互相拆台？目前是配比工程（炼丹），没有理论指导。

## 六、Token 经济学：长度、密度与模态不对等

> **对应你的问题**（这是你提得最具体、最工程的一组）："视频等非语言模态会占极长序列，而语言指令短得多；非语言模态的 token 该怎么给？怎么解决它太长？有必要用那么多 token 吗？和语言 token 数量不对等会有什么影响？需要为此做专门设计吗？"

逐个回答。

### 6.1 "非语言模态的 token 怎么给？"——把视觉切成 token 的两条路

回到第四节：要么**离散**（VQ 给每个 patch 一个码本 ID）、要么**连续**（ViT 给每个 patch 一个高维向量，经投影进语言隐空间）。无论哪条，一张图/一段视频都会被切成**大量** token：

- 一张 256×256 图，VQ 后约 **256** 个 token；高清图更多。
- 视频 = 图 × 帧数，token 数**爆炸**。一段几分钟的视频轻松上万 token，而你的指令可能就十几个字。

这就是你说的**严重不对等**：视觉 token 数 ≫ 语言 token 数。

### 6.2 "怎么解决它太长？"——压缩，但有根本权衡

Week 8 Part 2 已讲了具体手段，这里给你**一张全景**和**为什么它是个权衡**：

| 手段 | 机制 | 代价 |
| :--- | :--- | :--- |
| **空间池化 / Merging**（Qwen2-VL 的 2×2 合并） | 相邻 patch 特征合并，token 数 ÷4 | 损失细节（小目标、密集文字） |
| **Query 压缩**（Q-Former / Resampler，BLIP-2、Flamingo） | 用固定数量可学习 query 从视觉特征里"抽"出定长摘要 | 信息瓶颈，固定预算不适配复杂图 |
| **时序池化**（视频帧分组合并，如 4 帧一组） | 沿时间轴下采样 | 损失快速动态 |
| **可变 token / 动态分辨率**（Qwen 原生动态分辨率） | 按图像实际尺寸给 token，重要的多给 | 工程复杂（变长 batch 难） |

**根本权衡**：**压缩率 ↔ 保真度**。token 给少了省算力但"近视"（看不清细节、OCR/计数崩）；给多了看得清但算力爆炸、还稀释语言。**没有免费午餐。**

### 6.3 "有必要用那么多 token 吗？"——从信息密度看，多数是冗余的

直觉上**不必要**：视觉信号高度冗余，一张图的"语义"远没有上千个 token 那么多信息。问题在于**冗余和细节混在一起**——你不知道哪几个 patch 是关键（那行小字、那个红绿灯）。所以保守做法是"多给点别漏"，于是浪费。

前沿方向正是**任务自适应 token 预算**：简单图少给、复杂图（密集文档、细节问答）多给；甚至让模型**自己决定**要不要"放大看"某块区域。这与第七节的"用图思考"是连着的。

### 6.4 "数量不对等有什么影响？"——三个真实后果

1. **注意力被稀释**：上万视觉 token + 十几个文字 token 同处一个序列，语言指令的信号在注意力里被海量视觉 token "淹没"，模型容易"看了图忘了问题"。
2. **算力/显存成本由视觉主导**（连 [Week 6 推理优化]）：注意力是 O(L²)，prefill 成本和 KV cache 占用几乎全被视觉 token 吃掉。**视频 VLM 的推理瓶颈本质是个长序列工程问题**，而长序列里 99% 是视觉。
3. **位置编码的挑战**（连 Week 8 M-RoPE / TMRoPE）：极长且二维/三维（空间+时间）的视觉序列，逼出了 M-RoPE、TMRoPE 这类专门的位置编码——这正是"模态不对等"在位置编码上的回响。

> **结论**：**需要专门设计，而且这套设计是 VLM 工程的重头戏。** 视觉 token 的"码率"（用多少 token 表达多少视觉信息）没有理论最优解，目前全靠经验权衡。

> **开放问题**：是否存在"视觉的最优码率"理论——给定任务，多少 token 是信息论上够用的下界？没有。这是连接信息论与多模态架构的一块空白。

## 七、你可能还没考虑到的重要问题（补充）

> 你说"对这个领域了解可能还不够，可能有重要问题没考虑到"。以下五条是我认为同等重要、但你没点到的难点。第 (1) 条尤其关键——它是你整个"多模态没让模型更强"观察的**方法论地基**。

### (1) 评测危机：VLM 到底有没有在"看"？

这是最该补的一条。**很多 VLM benchmark，不看图也能答对。**

- 论文《Vision language models are blind》（arXiv 2407.06581, 2024）发现：VLM 在任何"睁眼人"都能秒答的初等视觉任务上大量失败（数线段交点、判断两圆是否相交……）。
- 大量 VQA benchmark 存在**语言先验捷径 / 数据泄漏**：模型靠常识猜答案，不依赖图像（"香蕉是什么颜色？"——不用看图）。
- 为对抗这点，NaturalBench（NeurIPS 2024）等用"两图两问、答案交替"的设计，让"瞎猜模型"拿不到分。
- 更深的根源——《Eyes Wide Shut?》（Tong et al., CVPR 2024）指出：**CLIP 类视觉编码器本身有系统性盲区**（CLIP-blind pairs：语义不同的两张图被 CLIP 编码成几乎一样的向量），这些盲区会**直接传染**给搭在它上面的 VLM。

> **为什么这条对你最重要**：你说"很多研究发现引入多模态没让模型更强"。这个观察成立的**前提**，是评测本身能可靠地测出"视觉是否带来增益"。而如果 benchmark 大量被语言先验污染，那么"加了视觉分数没涨"既可能是"视觉真没用"，也可能是"评测根本没在考视觉"。**先校准评测，才能讨论多模态有没有用。** 这是讨论一切 VLM 能力争议的地基。

### (2) 感知天花板 vs 推理天花板：VLM 常常"近视"

VLM 的失败常常不在推理，而在**感知**：OCR 小字、数物体个数、读图表精确数值、分辨细微差异——这些"低级"任务反而难。原因在第四节已埋下：视觉理解的训练信号是**从语言 loss 间接回传的**，很弱；加上 encoder（CLIP/SigLIP）的对比学习目标天生丢细节。所以要区分：模型答错，是**没看清**（感知天花板）还是**没想明白**（推理天花板）？二者的解法完全不同。

### (3) 数据问题：多模态没有"互联网级"的干净燃料

LLM 有近乎无限的高质量文本。VLM 的困境：① 高质量**图文配对/交错**数据稀缺得多；② 网络 caption 质量极差（alt-text 噪声大，要 DFN 一类方法过滤，见 Week 8 附录 A.1）；③ **多模态 scaling law 不清晰**——加多少视觉数据、配比多少、能换来多少能力，远没有 Chinchilla 那样的清晰规律。数据是 VLM 比 LLM 更被"卡脖子"的环节。

### (4) "用图思考"：多模态的 Test-Time Scaling（连 [Week 7]）

Week 7 讲的 test-time scaling（思维链、树搜索）几乎都在**语言空间**推理。前沿问题是：**能不能在视觉/像素空间里推理？** 比如 o3 类模型在 thinking 链中**裁剪、放大、圈画**图像再继续推理；或几何题里**画辅助线**。这是把"慢思考"从纯文本扩展到多模态的新方向，也回答了第六节的"任务自适应 token"——让模型自己决定"放大看哪里"。这是 2025–2026 最活跃的多模态推理前沿之一。

### (5) 通向具身：VLM 作为 VLA / 世界模型的基座（连你已有的专题）

VLM 不只是"聊天看图"的终点，它是**具身智能的基座**：在你已写的 [VLA 专题](视觉-语言-动作模型（VLA）.md) 里，VLM 被改造成输出**动作**而非文字；在 [世界模型专题](世界模型.md) 里，多模态生成能力被用来**预测世界的下一状态**。这给前面所有问题加了新约束——比如延迟（机器人要实时）、动作模态的 token 化（动作既不是文字也不是像素）。VLM → VLA 这条路，会把"理解-生成统一""token 经济学"这些问题推到更极端的场景。

## 八、收敛成一张图

把全文压成一张总表——**LLM 有的、VLM 多出来的难点、当前认知、是否开放**：

| 维度 | LLM 的情况 | VLM 多出来的难点 | 当前最好的认知 | 是否仍开放 |
| :--- | :--- | :--- | :--- | :--- |
| **本质关系** | 智能内核（推理载体） | 加了感知维度，但不天然增强推理 | 交集而非超集；感知↑推理未必↑ | 哲学层面开放 |
| **能力干扰** | 单模态无此问题 | 多模态税：语言能力退化 | 数据回灌/蒸馏可对冲，难根除 | ✅ 开放 |
| **架构** | 收敛到 Decoder-only + NSP | 原生 vs 模块化、原生内部又分裂 | 任务依赖，无统一架构 | ✅ 开放 |
| **建模目标** | NSP 一统理解与生成 | 视觉无无损 tokenizer；理解-生成撕裂 | 解耦/双目标缓解，无统一目标 | ✅ 强开放（圣杯） |
| **后训练** | RLHF/RLVR | 抗幻觉、grounding、视觉安全、护语言 | 需额外目标 + 配比工程 | ✅ 开放 |
| **Token 经济** | 文本无损固定 | 视觉超长、模态不对等、压缩 vs 保真 | 池化/重采样/动态分辨率权衡 | ✅ 开放（无最优码率理论） |
| **评测** | 较成熟（含可验证奖励） | 语言先验捷径、benchmark 泄漏、encoder 盲区 | vision-centric 评测在补 | ✅ 开放（地基性问题） |
| **数据** | 互联网级干净文本 | 图文数据稀缺、caption 噪声、scaling law 不明 | DFN 过滤、合成数据 | ✅ 开放 |

**回到最初的问题**——为什么顶级厂商仍主攻 LLM、为什么多模态没让模型一跃更强：

> 因为**智能的内核（推理）寄生在语言这个高密度符号系统上**，而视觉提供的是**正交的感知维度**而非更强的推理。把两者塞进一个模型，目前**收益（感知）和成本（多模态税、架构/建模/评测的全面不成熟）并存**。VLM 与 LLM 不是"谁强谁弱"，而是**两种性质不同的能力**，如何让它们在一个模型里"1+1≥2"而不是互相拆台——这正是当前所有未解难点的总和。
