# 世界动作模型（WAM）：统一世界建模与动作生成

*补充来源：阅读 LingBot-VA（蚂蚁集团，2026）和 DreamZero（NVIDIA，2026）论文时*

> **前置知识**：本文假设读者已了解 VLA 的基本架构（见 [VLA 专题](视觉-语言-动作模型（VLA）.md)）和扩散模型/Flow Matching 的基本原理（见 [扩散模型专题](扩散模型.md)）。理解 VLA 第十二节的"根本局限"是阅读本文的最佳前提——WAM 的每一个设计选择都是在回答那里提出的问题。

## 一、世界模型：为什么"想象未来"是控制的基础

在介绍 WAM 之前，需要先建立"世界模型"这个概念的直觉。

人类能做精确操作，不仅是因为手眼协调好，更重要的是**大脑在行动前会预演**：把鸡蛋放进袋子时，你不需要实际试验，脑子里就能预判"如果用力太猛，鸡蛋会碎"。这种"在脑中模拟世界如何响应动作"的能力，就是世界模型的本质。

**正式定义**：世界模型是描述"给定当前状态和执行的动作，世界下一时刻会变成什么"的函数：

$$s_{t+1} \sim p_\theta(s_{t+1} \mid s_t, a_t)$$

这被称为**正向动力学（Forward Dynamics）**——与"静力学"（世界现在是什么样）相对，动力学描述的是世界状态如何随动作演化。

**有了世界模型能做什么？**

- **推理时规划**：在执行前想象多条路径，选择预测结果最好的那条
- **闭环纠偏**：每步执行后，对比"预测的下一帧"和"真实的下一帧"，立刻发现偏差并修正
- **数据高效的跨任务迁移**：物理规律是通用的，在一个任务上学到的"推物体会发生什么"可以直接迁移

VLA 没有这个能力——它直接学 $\pi(a \mid o)$，知道"此刻该做什么"，但不建模"做了之后世界会怎样"。

## 二、概念定位：WAM 在具身智能图谱中的位置

理解 WAM 最直接的方式是三角对比：

```
纯世界模型（World Model）
  学习：p(s_{t+1} | s_t, a_t)
  能做：预测"执行动作后世界变成什么"
  不能：直接输出机器人控制指令

纯 VLA / 策略网络（Policy）
  学习：π(a_t | o_t, 指令)
  能做：从当前观测直接映射到动作
  不能：理解"动作导致什么物理后果"

WAM（World Action Model，世界动作模型）
  学习：p(video, action | 历史观测, 指令)
  能做：同时预测未来世界状态 + 生成可执行动作
  关键：两者共享同一套表示，联合优化
```

WAM 的核心洞见：**"理解因果动力学"和"生成正确动作"不是两件独立的事，本质是同一件事的两面。** 真正理解了"推这个物体它会滑到哪里"，自然就知道"应该用多大力推才能让它停在目标位置"。

WAM 一词由 **JOWA**（ICLR 2025）首次正式提出，最初用于 Atari 游戏的离线强化学习。**DreamZero**（NVIDIA，2026）将这一框架带入视频扩散 + 机器人操作的场景，给出了迄今最完整的定义：

> *"WAMs learn physical dynamics by predicting future world states and actions, using video as a dense representation of how the world evolves."*

## 三、动机：VLA 的四道墙

VLA 专题第十二节详细讨论了 VLA 的根本局限，这里做简短的映射，说明 WAM 的每个设计选择是在回答哪个问题：

| VLA 的局限 | 根本原因 | WAM 的对应设计 |
|-----------|---------|--------------|
| 执行出错无法恢复（分布偏移） | 只学"专家怎么做"，不建模"做了会发生什么" | 学习正向动力学，推理时持续预测-观察-纠偏 |
| 语义理解与精控争用表示空间 | 同一参数做两件性质不同的事 | 世界模型负责语义+物理预测，动作头只从预测结果推导动作 |
| 无法利用无标注视频 | 必须有动作标签才能训练 | IDM 解耦：Stage 1 不需要动作标注 |
| Action Chunk 执行期间无反馈 | 动作生成完就固定了 | 每步用真实帧替换预测帧，实现闭环 |

这四道墙再加上触觉/力控的缺失（WAM 同样尚未解决），构成了当前具身智能面临的核心工程挑战。WAM 直接回应前三条，触觉/力控是正交的独立问题（见 VLA 专题 12.5 节）。

## 四、IDM：WAM 的核心数据解耦机制

在介绍具体实现范式之前，需要先理解 WAM 的数据策略核心：**逆动力学模型（Inverse Dynamics Model, IDM）**。

### 4.1 正向 vs 逆向动力学

```
正向动力学（Forward Dynamics）：p(s_{t+1} | s_t, a_t)
  "如果我现在做动作 a，世界会变成什么样？"
  → 世界模型就是这个

逆动力学（Inverse Dynamics）：p(a_t | s_t, s_{t+1})
  "是什么动作，让世界从状态 s_t 变成了 s_{t+1}？"
  → 不直接预测动作，先预测世界变化，再反推动作
```

### 4.2 为什么这个分解有巨大价值

IDM 的关键在于**训练数据的解耦**：

- **正向预测（学"世界会怎么变"）**：不需要动作标注——任何视频都可以用，YouTube 视频、家庭录像、自然纪录片……
- **逆动力学（从"世界变化"推"什么动作"）**：只有这部分需要机器人演示数据（带动作标注）

机器人演示数据有限（几万到几十万条）且采集成本极高（需要专业人员遥操作），但无标注视频几乎无限。IDM 让两类数据各司其职，大幅放大数据利用率。

LingBot-VA 的两阶段公式明确体现了这个分解：

$$\underbrace{o_{t+1} \sim p_\theta(\cdot \mid o_{\leq t})}_{\text{Stage 1：视觉动力学（无需动作标注）}} \qquad \underbrace{a_t \sim g_\psi(\cdot \mid o_t,\ o_{t+1})}_{\text{Stage 2：逆动力学（需要动作标注，但量少）}}$$

**重要说明**："可利用无标注视频"的优势对 Stage 1 成立；Stage 2 仍然需要带标注的机器人演示，但由于 Stage 1 已提供了强大的视觉动力学表示，Stage 2 所需标注数据量大幅减少——LingBot-VA 每项任务仅需约 50 条演示。

## 五、两种主流实现范式

本节的两种范式都是"**自回归处理上下文 + FM/扩散生成连续动作**"这一混合结构的具体实例化——只是在"AR 处理什么"和"扩散生成什么"这两个维度上有所不同。这个混合结构的动机和设计逻辑在 VLA 专题 6.5 节有完整解释，本节直接从两种范式的具体实现展开。

### 5.1 范式一：视频-动作交织自回归（LingBot-VA）

**核心思想**：将视频 token 和动作 token 交织成**一条统一的自回归序列**，用因果注意力掩码处理，物理因果性从架构层面得到保证。

**训练的两个阶段，序列结构不同**：

```
Stage 1 训练——无标注视频，无动作 token：
  [o₁,  o₂,  o₃,  o₄,  ...]
  目标：给定历史帧，预测下一帧 → 不需要任何动作标注

Stage 2 训练——带标注演示，序列为"帧-帧-动作"交织：
  [o₁, o₂, a₁, o₃, a₂, o₄, a₃, ...]
       ↑    ↑
   下一帧  动作（由相邻两帧推导：IDM）

  → 预测 a₁ 时，模型可以看到 o₁ 和 o₂，
    即知道"世界发生了什么变化"，再推断"是什么动作导致了这个变化"
  → 这正是逆动力学（IDM）的本意：a_t ~ g(o_t, o_{t+1})

因果注意力掩码（两阶段均适用）：
  每个 token 只能看到序列中位置更靠前的 token
  → 预测 a₁ 时可以看 o₁、o₂，但看不到 o₃ 以后
  → 物理因果性从架构层面得到保证
```

**两步走的推理流程**：Stage 2 的序列设计意味着推理时也是两步：
1. 世界模型先生成 $o_{t+1}$（"下一时刻会变成什么"）
2. IDM 再从 $(o_t, o_{t+1})$ 推导 $a_t$（"是什么动作导致了这个变化"）

每输出一个动作需要先完成一次帧预测。这和 DreamZero 的"同时预测"形成对比。

**为什么两阶段分开训练**：Stage 1 与动作无关，可用任意视频大规模训练物理先验；Stage 2 需要动作标签，但视觉理解已在 Stage 1 学好，只需少量带标注演示即可。

**闭环控制机制**（解决 VLA 反应性缺口的关键）：

```
执行一个 action chunk 后：
  模型"想象"的预测帧（下一时刻应该是什么）
       → 被真实摄像头观测替换掉
       → 真实帧注入 KV Cache

效果：
  1. 每步都用真实观测校正，误差不累积
  2. 全部历史缓存在 KV Cache，长时记忆问题得到解决
  3. 视觉偏差立刻被检测并反映在下一步决策中
```

动作以**连续潜在向量**表示，通过 Flow Matching 框架迭代去噪生成（见附录 A.1）。

### 5.2 范式二：视频-动作联合扩散（DreamZero）

**核心思想**：用**同一个 DiT 主干**在同一个去噪时间步内，同时对视频 latent 和动作向量去噪——两者在单次前向传播中联合生成。

**数学分解**：

$$p(\text{video}, \text{action} \mid \text{context}) = \underbrace{p(\text{video} \mid \text{context})}_{\text{世界模型}} \times \underbrace{p(\text{action} \mid \text{video}, \text{context})}_{\text{逆动力学模型}}$$

虽然数学上写成链式分解，但实现上是**单次前向传播同步完成**：

```
输入到 DiT：
  [含噪视频 latent z_{t_k}]  +  [含噪动作向量 a_{t_k}]
  （同一个去噪时间步 t_k，两者噪声程度完全一致）
             │
             ▼
       DiT 主干（共享权重，单次前向传播）
             │
         ┌───┴───┐
         ▼       ▼
     视频解码  动作解码
     (未来帧)  (控制指令)

训练损失：联合流匹配损失
  Loss = ‖预测速度 - 真实速度‖²
  真实速度 = [z₁ - z₀, a₁ - a₀]（视频和动作拼成的联合速度向量）
```

**共享去噪时间步是关键**：视频 latent 和动作向量始终在相同噪声程度下处理，梯度可以在一次反向传播中同时回传，实现真正的联合优化，而非两个独立模型的串联。

**DreamZero-Flash（实时部署方案）**：标准扩散模型对视频需要多步去噪，速度无法满足机器人 7Hz 以上控制频率。DreamZero 的解法是**解耦两者的噪声调度**：视频走多步去噪（保质量），动作用 Beta 分布调度实现**单步去噪**（保速度）。结合 KV Cache 与异步执行，在双 GB200 上实现 7Hz 闭环控制（150ms/chunk）。

### 5.3 两种范式对比

| 维度 | LingBot-VA（交织自回归） | DreamZero（联合扩散） |
|------|------------------------|---------------------|
| **架构基座** | 视频生成预训练模型（自回归） | Wan2.1-I2V（14B，DiT 扩散） |
| **每步预测流程** | 两步：先预测 $o_{t+1}$，再从 $(o_t, o_{t+1})$ 推 $a_t$ | 一步：视频 latent 与动作向量同一次前向传播同时输出 |
| **因果机制** | 因果注意力掩码（架构强制） | 自回归 chunk，chunk 内联合生成 |
| **闭环设计** | 真实帧替换 + KV Cache | 真实帧替换 + KV Cache |
| **实时性** | Flow Matching 步数少，较快 | 需 DreamZero-Flash 优化才达标 |
| **零样本泛化** | 预训练后直接使用 | 已见任务 62.2%，未见任务 39.5% |

## 六、JOWA：WAM 概念的起点

机器人 WAM 的思想最早在游戏 AI 领域被系统验证。

**JOWA（Joint World and Action Model，ICLR 2025）** 的出发点是 Atari 游戏的离线强化学习：给定一组人类游戏录像（状态-动作序列），不通过在线交互，直接学到高质量的游戏策略。

JOWA 在**同一个 Transformer 骨干**上设置两个预测头：

```
                    ┌───────────────────────────────┐
输入：(s_t, a_t) →   │   共享 Transformer 骨干        │
                    └──────────┬──────────┬─────────┘
                               ▼          ▼
                   下一状态预测 s_{t+1}   Q 值估计 Q(s_t, a_t)
                   （世界模型头）         （策略头）
```

两个头共享底层表示，训练损失是两者之和：

$$\mathcal{L} = \lambda_{\text{world}} \cdot \mathcal{L}_{\text{next-state}} + \lambda_Q \cdot \mathcal{L}_{\text{TD}}$$

**为什么联合优化有效**：两个目标互相正则——下一状态预测避免了 Q 学习的过拟合；Q 学习的价值信号又引导世界模型聚焦于"对决策有用"的状态变化，而不仅是像素级重建精度。

JOWA 在 Atari 100K 基准（每个游戏仅 100K 步数据）上显著超越了当时最好的离线 RL 方法，验证了"共享表示、联合训练"这一核心思路的有效性。

机器人领域的 WAM（LingBot-VA、DreamZero）直接继承了这个核心架构思路——只是把游戏状态换成了视频帧，把 Q 值头换成了动作生成头。

## 七、WAM 的数据优势

这是 WAM 范式最重要的工程价值。

| | VLA | WAM Stage 1 | WAM Stage 2 |
|--|-----|------------|------------|
| **数据类型** | 视频 + 动作标注 | 任意视频（无需标注） | 视频 + 动作标注 |
| **数据规模** | 几万～几十万条 | 理论无限 | 与 VLA 相当，但需求更少 |
| **采集成本** | 高（专业遥操作） | 低（网络爬取即可） | 高，但量少 |
| **代表来源** | OXE、BridgeData | YouTube、家庭视频 | LingBot-VA 每任务 ~50 条 |

DreamZero 的跨具身迁移效率直观体现了这个优势：
- 基础模型在 AgiBot G1 数据集（~500 小时）上训练
- 迁移到新机型只需 **30 分钟 play data（55 条轨迹）**
- 迁移后零样本泛化能力保持，相对提升 >42%

**极少量标注数据即可跨具身迁移**，对于实际部署场景（每台新机器人都需要采集大量数据的问题）有直接的商业价值。

## 八、泛化能力：WAM 为什么能超越 VLA 的泛化上限

VLA 本质上是**判别模型**——学习"状态 → 动作"的直接映射，本质上是对训练任务主导行为的记忆，遇到分布外场景很快失效。

WAM 通过**生成式目标**（预测未来世界状态）学习因果关系：

```
VLA 学到的：
  "看到桌上有杯子且指令是'抓杯子' → 输出关节角度 [x, y, z, ...]"
  → 新场景、新物体，这个映射直接失效

WAM 学到的：
  "从 o_t 到 o_{t+1}，末端执行器向左移动了 Δx cm"
  → 理解了"什么视觉变化对应什么运动"，可推广到没见过的场景
```

DreamZero 的实验数据直接验证：在**未见任务**（zero-shot）上，DreamZero 成功率 39.5%，最佳预训练 VLA 基线仅 16.3%，相差超过两倍。

泛化优势的来源不只是架构，也包括**预训练数据的广度**——WAM Stage 1 吃的是人类活动视频，其中包含物体运动、物理交互、手部操作等物理规律，这些规律跨任务、跨物体是共通的。

## 十、与其他世界模型路线的关系

WAM 不是具身智能里唯一的世界模型路线，放在更大的图谱里：

| 路线 | 代表工作 | 世界模型形态 | 动作接口 | 关键区别 |
|------|---------|------------|---------|---------|
| **潜空间世界模型** | DreamerV3 | RSSM，潜空间动力学预测 | 策略在想象空间里训练（RL） | 紧凑高效，可大量想象采样；但与真实像素脱钩 |
| **视频仿真器** | UniSim、IRASim | 视频生成，像素输出 | 在生成的虚拟视频上训练独立策略 | 视觉逼真，但策略提取是额外的独立步骤 |
| **WAM** | LingBot-VA、DreamZero | 视频生成 + 动作联合 | 动作与视频从同一模型输出 | 端到端统一，直接闭环控制，无接口成本 |

WAM 可以理解为"视频仿真器路线"的进化版——把原来分离的"视频预测"和"策略提取"两步合并成一个模型，消除了中间的接口成本，代价是参数量更大、训练更复杂。

## 十一、当前挑战与开放问题

**计算成本**：视频扩散主干（如 Wan2.1-I2V，14B 参数）的推理成本远高于纯 VLA。DreamZero 在双 GB200 上才勉强达到 7Hz 控制频率，在普通机器人硬件（单张消费级 GPU）上部署 WAM 是当前最紧迫的工程挑战。

**动作频率与视频帧率的错配**：机器人控制通常需要 30–100Hz，视频帧率通常是 5–30FPS。DreamZero 的采集方案（30Hz 控制，5FPS 视频采样）是工程妥协——高频动作用 Action Chunking 打包，但帧间细粒度反馈因此丢失。

**精细接触操作**：WAM 继承了视频生成模型的局限——接触力、摩擦等物理量在像素空间里没有显式表示。插管、拧螺丝等精细操作的 WAM 方案尚不成熟（与 VLA 的触觉/力控缺失问题重叠）。

**联合训练稳定性**：世界模型预测损失和动作生成损失在量级和梯度方向上可能不一致，需要精心调节损失权重和训练课程。JOWA、DreamZero 均报告了联合训练比单目标训练需要更多调参经验。

**无标注视频的物理知识迁移边界**：WAM 假设"人类活动视频蕴含足够的物理动力学知识"。对于手部操作类任务（与人类行为相似）这一假设合理；对于工业机械臂的特定运动模式（与人类操作差异大），迁移效果存疑。当前缺乏系统实验来验证这个边界在哪里。

**长时规划的误差累积**：真实帧替换机制缓解了短时误差，但对于几分钟量级的长任务，世界模型的累积预测误差仍会影响执行质量。WAM 框架内的可靠长时规划仍是开放问题。

### A.1 Flow Matching 在 WAM 中的角色

> 关联正文：**5.1 节 → LingBot-VA 动作生成**

LingBot-VA 的动作生成不是 next-token 预测，而是 Flow Matching。简要回顾：

Flow Matching 学习一个"速度场" $v_\theta(x_t, t)$，描述从噪声 $x_0 \sim \mathcal{N}(0, I)$ 到目标分布 $x_1$（目标动作）的流动路径：

$$\frac{dx_t}{dt} = v_\theta(x_t, t)$$

训练目标是让预测速度场尽量接近真实速度场（两点之间线性插值路径的方向向量）。推理时，从随机噪声出发，沿预测速度场积分 $k$ 步（通常 3–10 步），得到生成的动作。

与扩散模型（DDPM）相比，Flow Matching 的插值路径是直线，梯度方差更小、训练更稳定、推理步数更少。DreamZero 的联合生成损失也基于 Flow Matching 框架：

$$\mathcal{L}_{\text{FM}} = \left\| v_\theta\left(\begin{bmatrix} z_t \\ a_t \end{bmatrix}, t\right) - \begin{bmatrix} z_1 - z_0 \\ a_1 - a_0 \end{bmatrix} \right\|^2$$

其中 $z$ 是视频 latent，$a$ 是动作向量，两者拼在一起作为联合速度向量训练。详细介绍见 [扩散模型专题](扩散模型.md)。

### A.2 KV Cache 在闭环控制中的作用

> 关联正文：**5.1 节 → LingBot-VA 闭环控制机制**

LingBot-VA 的闭环控制依赖 KV Cache 实现高效的历史持久化。KV Cache 是 Transformer 推理的标准优化：计算过的历史 token 的 Key/Value 矩阵被缓存起来，新 token 的注意力计算直接复用，避免重复计算。

在 LingBot-VA 中，这个机制被用来：
1. **持久化全部历史**：每次执行 action chunk 后，把新的真实观测帧编码为 token 并追加到 KV Cache，历史不会因为"块切换"而丢失
2. **真实帧替换**：预测的下一帧只是占位符，KV Cache 更新时用真实摄像头帧覆盖，实现对现实世界的持续校正

这使得 LingBot-VA 的有效上下文长度随任务执行时间线性增长，但推理成本仅增加新 token 的计算量，不需要对历史重新计算。
