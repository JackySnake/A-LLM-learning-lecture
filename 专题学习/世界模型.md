# 世界模型（World Model）：预测先于控制

*补充来源：机器人/具身智能调研，与 WAM 和 VLA 专题互为背景知识*

> **前置知识**：本文不假设特定前置，但与 [VLA 专题](视觉-语言-动作模型（VLA）.md) 和 [WAM 专题](世界动作模型（WAM）.md) 紧密关联。建议阅读顺序：本文（世界模型通论）→ VLA 专题 → WAM 专题。

## 一、世界模型是什么

**世界模型**的核心定义是一个**动力学预测模型**：给定当前状态 $s_t$ 和动作 $a_t$，预测下一时刻的状态 $s_{t+1}$。

$$\text{世界模型：} p(s_{t+1} \mid s_t, a_t)$$

对比 VLA（视觉-语言-动作模型）：

| | VLA（反应式策略） | 世界模型（预测模型） |
|---|---|---|
| **核心能力** | 感知 → 直接输出动作 | 给定动作 → 预测未来状态 |
| **学习目标** | 动作监督（模仿专家） | 状态预测（理解物理规律） |
| **推理时行为** | 看一帧，输出一个动作 | 可在内部"想象"多条轨迹再决策 |
| **类比** | 老司机的条件反射 | 沙盘推演后再行动 |

> **"世界模型"不是一个固定架构**：这个词在不同语境下指代不同的东西。Yann LeCun 的理论框架里指"联合嵌入预测架构（JEPA）"；强化学习领域里指"环境动力学模型（RSSM/DreamerV3）"；视频生成领域里指"可交互视频生成器"。共同点只有一个：**预测世界状态的演变**。

## 二、两类世界模型：潜空间 vs 像素空间

在具体实现上，世界模型分为两大流派，各自有根本性的设计取舍。

### 2.1 潜空间世界模型

**核心思路**：不在像素空间预测，而是先把图像压缩成低维潜变量（latent），再在潜空间里学习动力学。

$$s_t = \text{Encoder}(o_t) \quad \rightarrow \quad s_{t+1} = f(s_t, a_t) \quad \rightarrow \quad \hat{o}_{t+1} = \text{Decoder}(s_{t+1})$$

其中：$o_t$ 是 $t$ 时刻的**原始观测**（摄像头像素图像，高维）；$s_t$ 是编码后的**潜在状态**（低维压缩向量，世界模型的实际工作空间）；$\hat{o}_{t+1}$ 中的帽子 $\hat{\cdot}$ 表示"预测值"，即解码器把预测潜态 $s_{t+1}$ 还原回像素得到的**预测图像**。

**代表**：**DreamerV3**（DeepMind）中的 RSSM（Recurrent State Space Model）

**优点**：
- 潜空间维度低（几百维 vs 像素空间的几百万维），预测计算量小
- 可以在"想象世界"里快速采样成千上万条轨迹用于策略训练
- 训练稳定，样本效率高

**缺点**：
- 潜空间是抽象的，不直接对应可观测的像素——世界模型的"想象"没法直接拿来看
- 编码器的信息损失可能丢弃控制所需的细节
- 与视觉语言模型（VLM）的集成不直接，因为 VLM 在像素/patch 级别工作

### 2.2 像素空间世界模型（视频生成模型）

**核心思路**：直接用视频生成模型预测未来帧。给定当前画面和动作，生成未来的视频帧序列。

$$\hat{o}_{t+1}, \hat{o}_{t+2}, \ldots = \text{VideoModel}(o_{\leq t}, a_t, \text{指令})$$

**代表**：UniSim（Google）、IRASim（字节 + 北大）、Wan2.1-I2V（阿里，被 DreamZero 用作骨干）

**优点**：
- 输出是真实画面，人和下游策略都可以直接使用
- 可以充分利用海量互联网视频预训练（不需要特定格式）
- 与 VLM / LLM 的集成天然——同样在 patch/pixel 级别工作

**缺点**：
- 计算量巨大（视频生成本身就是重任务）
- 像素级细节的物理一致性难以保证（视觉上合理但物理上可能错误）
- 推理速度慢，实时控制需要专门优化（如 DreamZero-Flash）

### 2.3 两类的比较

| 维度 | 潜空间世界模型 | 像素空间世界模型 |
|------|-------------|--------------|
| **计算成本** | 低（潜空间小） | 高（视频生成昂贵） |
| **与 VLM 集成** | 不直接 | 天然兼容 |
| **可解释性** | 低（抽象潜变量） | 高（可视化未来帧） |
| **物理精度** | 取决于编码器质量 | 难以保证像素级物理一致 |
| **数据利用** | 需要状态-动作标注 | 可利用无标注视频 |
| **代表** | DreamerV3、RSSM | UniSim、Wan2.1、DreamZero |

WAM（世界动作模型）采用的是**像素空间路线**，直接在视频帧级别联合建模世界状态和动作。

## 三、具身智能的四条技术路径

世界模型在具身智能里有四条主要应用路径，从"分离辅助"到"深度统一"递进。

### 路径一：Model-based RL（基于模型的强化学习）

**思路**：先用少量真实交互训练一个潜空间世界模型，再让策略在世界模型的"想象空间"里进行大量试错训练，不需要每次都让真实机器人去碰撞。

$$\text{真实环境（少量）} \rightarrow \text{世界模型} \rightarrow \text{想象空间（大量采样）} \rightarrow \text{策略}$$

**代表**：**DreamerV3**（DeepMind）——在压缩潜空间里学习动力学，策略完全在"梦境"中训练，最终迁移到真实任务。在 Atari、连续控制、机器人等多个领域用统一架构达到 SOTA。

**优势**：样本效率极高，真实交互数据少。  
**劣势**：世界模型误差累积（"模型谬误"），复杂任务中想象轨迹可能与现实偏差过大。

### 路径二：推理时规划（Test-time Planning）

**思路**：世界模型不参与训练，而是在**推理时**作为"仿真器"——机器人执行动作前，先在世界模型里模拟多条候选轨迹，选择预测结果最好的那条再执行。

$$\text{采样多条动作序列} \xrightarrow{\text{世界模型推演}} \text{预测各序列的未来状态} \xrightarrow{\text{选优}} \text{执行最佳动作}$$

**代表方法**：**MPC（模型预测控制）**、**MPPI**——经典控制领域的方法，被神经网络世界模型赋予了新生命。

**优势**：可以做长程规划，在新任务上泛化更好（不需要针对每个任务重新训练策略）。  
**劣势**：推理时需要大量采样，计算开销高，实时控制有延迟压力。

### 路径三：视频生成仿真器

**思路**：把**可交互视频生成模型**直接当作具身仿真器，输入当前帧 + 动作，输出未来视频帧。机器人策略在这个"视频仿真器"里收集训练数据，再用这些数据训练 VLA 策略。

$$\text{当前帧} + \text{动作} \xrightarrow{\text{视频生成世界模型}} \text{未来视频} \xrightarrow{\text{作为训练数据}} \text{VLA 策略}$$

**代表**：UniSim（Google）、IRASim（字节+北大）、Genesis（多机构，2025）

**优势**：可大规模生成合成数据，突破真实机器人数据稀缺的瓶颈。  
**劣势**：生成视频与真实物理存在"感知漂移"——视觉合理但物理不准确，策略在真实场景中可能失效。

### 路径四：WAM（世界-动作统一建模）

**思路**：不把世界模型和策略分成两个阶段，而是在**同一个模型**里同时学习"未来世界状态预测"和"动作生成"，两者共享表示并联合优化。

这是路径一（Model-based RL）和路径三（视频仿真器）的融合与升华：不再是"先用世界模型采样，再训策略"的两阶段，而是世界预测本身就是动作生成的一部分。

**代表**：LingBot-VA（蚂蚁集团，2026）、DreamZero（NVIDIA，2026）

**这是当前最前沿、研究热度最高的路径**，详见 [WAM 专题](世界动作模型（WAM）.md)。

## 四、LeCun 的世界模型愿景：JEPA 框架

Yann LeCun（Meta AI 首席 AI 科学家）在 2022 年提出了一个与主流扩散/AR 路线不同的世界模型框架：**JEPA（Joint Embedding Predictive Architecture，联合嵌入预测架构）**。

### 4.1 JEPA 的核心思路

JEPA 的出发点是批判生成式方法（扩散模型/VAE）的一个根本问题：**预测像素太难，大量计算浪费在预测"不重要的细节"上**（比如纹理噪声、光照细节），而真正对决策有用的高层语义信息反而没有被充分利用。

JEPA 的解法是：**不预测像素，预测表示**。

```
JEPA 架构：
  上下文编码器（Context Encoder）：s_x = f(x)，编码当前观测
  目标编码器（Target Encoder）：s_y = f(y)，编码目标（未来状态）
  预测器（Predictor）：s_ŷ = g(s_x, z)，在表示空间预测目标

  训练目标：最小化 ‖s_ŷ - s_y‖²
  （在表示空间里预测，不解码回像素）
```

关键：目标编码器是**指数移动平均（EMA）的上下文编码器**（类似 BYOL 的思路），防止模型学到退化的表示（把所有东西映射到同一个向量）。EMA 编码器的原理、stop-gradient 设计和防崩塌机制在 **[自监督表征学习专题](自监督表征学习.md)** 的"动量蒸馏"和"教师-学生"两节中有完整介绍。

### 4.2 JEPA 与生成式世界模型的对比

| | JEPA | 生成式（扩散/VAE） |
|--|------|-----------------|
| **预测目标** | 高层表示向量 | 像素/潜变量（可重建为图像） |
| **计算效率** | 高（不需要解码回像素） | 低（像素重建计算量大） |
| **可视化** | 不能直接看"预测的未来"| 可以生成可视化的未来帧 |
| **下游灵活性** | 更抽象，需要下游头 | 输出可直接用于多任务 |
| **物理精度** | 侧重高层语义，物理细节弱 | 像素级细节，但可能不物理一致 |

### 4.3 JEPA 在实践中的位置

I-JEPA（2023）在图像自监督表征上取得了很好的效果，V-JEPA（2024）扩展到视频。但 JEPA 在机器人/具身智能领域的直接应用仍然有限——当前主流的 WAM 工作（LingBot-VA、DreamZero）都选择了像素/视频空间的生成式路线，而非 JEPA 的表示预测路线。

这并不意味着 JEPA 不重要，更可能是**两条路线在不同任务上各有优势**：JEPA 在纯视觉表示学习上效率更高，而具身控制任务需要可视化的未来帧来做 IDM 和闭环校正。

> **📖 关联专题：[自监督表征学习](自监督表征学习.md)** 介绍了 JEPA 所属的对比学习/自预测学习大家族，以及 MAE、CLIP、BYOL 等方法的完整演进。

## 五、世界模型与 VLA / WAM 的关系

这三个概念的层次关系如下：

```
感知输入（图像/语言）
        │
        ▼
  ┌─────────────┐
  │  世界模型    │  ← 理解"物理规律"：这个动作会导致什么
  │（预测未来）   │
  └──────┬──────┘
         │ 预测结果 / 规划信息
         ▼
  ┌─────────────┐
  │  策略模型    │  ← 决策"做什么动作"
  │（VLA 或 RL） │
  └─────────────┘
         │
         ▼
      执行动作
```

**三条技术路线的关系**：

- **VLA（纯策略）**：只有下层的策略模型，没有上层的世界模型，直接学"看到什么 → 做什么"
- **分离式世界模型**（路径一至三）：上层世界模型和下层策略是两个独立训练的模型，通过接口交互
- **WAM**：把上层和下层合并成一个模型，世界预测和动作生成共享表示、联合优化，彻底消除接口成本

这个演化方向类似于自动驾驶领域从"模块化方案"走向"端到端方案"的过程——不是彻底否定模块化，而是认识到**模块间的信息损失和接口开销**是性能天花板。

## 六、自动驾驶：世界模型落地最早的领域

自动驾驶是世界模型落地最成熟的领域，对具身机器人有强烈的参考价值——两者面临的核心挑战几乎一致：长尾场景稀缺、仿真与现实的感知漂移、闭环评测困难。

### 6.1 自动驾驶的世界模型在做什么

自动驾驶面临一个核心挑战：**长尾场景极其罕见**（路上突然窜出一只狗），真实数据永远收集不完。传统方案是用 CARLA 这类游戏引擎仿真，但游戏引擎与真实道路之间有明显的"感知漂移"，在仿真里训练好的模型搬到真实场景往往失效。

世界模型的方案：**用真实驾驶视频训练视频生成模型**，让它学会"给定当前场景 + 车辆行为，预测接下来的道路视频"。生成的场景在视觉分布上与真实数据无差别，可以无限合成长尾场景。

```
真实驾驶视频（大规模）
        │ 训练
        ▼
  自动驾驶世界模型
  输入：当前多路摄像头帧 + 自车行为（转向/加速/制动）
  输出：未来几秒的多路摄像头视频
        │
        ▼
  合成长尾场景 → 训练感知 / 规划模型
```

### 6.2 代表模型

**GAIA-1（Wayve，2023）**

- Wayve 是一家英国自动驾驶公司，GAIA-1 是其发布的生成式世界模型
- 输入：历史视频帧 + 文本描述（如"前方出现行人"）+ 自车动作
- 输出：生成未来驾驶视频，场景符合物理规律且视觉真实
- 关键能力：可以用**自然语言控制生成场景**，比如生成"夜晚下雨、前方有急刹车的场景"

**NWM（Neural World Model，蔚来，2024）**

- 核心设计：不仅预测自车视角的未来视频，还对**周围其他车辆的行为**建模（"他车动力学"），这对城市复杂路口的规划尤为重要
- 训练数据来自蔚来车队积累的大规模真实道路数据

**DriveDreamer / DriveX / Vista 等（学术界，2024）**

- 国内多个团队发布的同类工作，在生成质量和可控性上持续迭代
- 共同方向：提高对**稀有场景**（暴雨、逆光、施工路段）的生成能力

### 6.3 自动驾驶 vs 机器人世界模型

| | 自动驾驶世界模型 | 机器人具身世界模型 |
|---|---|---|
| **"世界"是什么** | 道路、车流、行人 | 桌面、物体、机械臂 |
| **动作空间** | 转向角、油门、制动（低维） | 关节角度、末端位姿（中维） |
| **主要用途** | 合成长尾驾驶场景 | 合成操作演示数据 / 规划 |
| **成熟度** | 较成熟，已在量产车上部署 | 仍处于研究阶段 |
| **核心难点** | 多智能体交互预测（他车行为） | 接触动力学（抓取、推拉的物理精度） |

> 两个领域最大的共同挑战是**因果一致性**：生成的视频在视觉上真实，但物理上不一定正确（物体穿模、光影矛盾）。如何让世界模型真正理解物理因果，而不只是"看起来像"，是当前核心开放问题。

## 七、自动驾驶系统全貌、数据闭环与数据飞轮

### 7.1 自动驾驶系统架构演进

**传统模块化方案（2015–2022 主流）**

各模块独立训练，输出传给下一模块：

```
传感器原始数据（摄像头 + LiDAR + 毫米波雷达）
        │
        ▼
  【感知】检测周围车辆/行人/车道线 → 输出结构化地图
        │
        ▼
  【预测】周围车辆/行人接下来会去哪？
        │
        ▼
  【规划】我应该走哪条轨迹？（全局路线 + 局部避障）
        │
        ▼
  【控制】把轨迹转成方向盘角度 + 油门 + 刹车指令
```

核心问题：**模块间信息损失不可逆**——感知输出的结构化数据丢失了大量原始细节，各模块优化目标不一致，整体不一定最优。

**端到端方案（2023 起主流转向）**

直接从原始传感器输入 → 输出控制指令，中间不做显式拆分。信息无损传递，但黑盒、难调试。蔚来和小鹏目前都在向**端到端 + 世界模型**的组合演进。

### 7.2 世界模型在系统中扮演的角色

世界模型不是"驾驶策略"本身，而是一个**配套基础设施**，解决三个问题：

| 问题 | 没有世界模型 | 有世界模型 |
|---|---|---|
| 长尾场景数据稀缺 | 靠游戏引擎仿真（感知漂移大） | 用真实数据训练的世界模型生成 |
| 策略评测要上真车 | 开环回放（不能测实际反应） | 世界模型里跑闭环仿真 |
| 规划时不知道后果 | 靠规则或历史统计 | 世界模型"想象"各决策的未来 |

> **开环 vs 闭环评测**：**开环（Open-loop）**回放录像，比较模型预测轨迹和人类实际轨迹的差距——快且便宜，但模型的错误决策不会影响后续观测，测的是"预测准不准"而非"真实驾驶好不好"。**闭环（Closed-loop）**让模型的每个动作都改变环境状态，更真实，但需要仿真器。世界模型的核心价值之一正是提供**视觉分布接近真实**的闭环仿真环境。

### 7.3 蔚来 NWM vs 小鹏端到端

**蔚来 NWM（Neural World Model）**

- 输入：当前多路摄像头（全车环视）+ 自车未来行为 + 周围车辆行为
- 输出：未来 N 秒的多路摄像头视频
- 核心亮点：对**他车动力学**建模——"如果旁边那辆车突然变道，我的摄像头会看到什么"
- 用途：① 生成危险场景合成训练数据；② 作为闭环仿真器评测驾驶策略

**小鹏端到端 + 世界模型**

分两层：BEV 感知 + 神经网络规划器直接从多摄像头输出轨迹（取消手写规则规划器）；世界模型生成反事实场景（"如果当时没有减速会怎样"）扩充训练数据。小鹏的优势在于积累了大量**中国特色长尾场景**（无保护左转、电动摩托车穿插、非标路口）。

### 7.4 数据闭环（Data Closed Loop）

从数据采集到模型更新再回到车辆的完整系统工程循环：

```
量产车队上路
    │ 采集原始传感器数据（TB/天级别）
    ▼
自动化数据挖掘
    │ 自动识别"模型表现差的片段"（接管事件、异常刹车等）
    ▼
场景筛选 + 标注
    │ 人工/半自动标注，或用世界模型生成场景变体
    ▼
模型训练
    │ 用新数据更新感知/规划模型
    ▼
仿真评测（开环 + 闭环）
    │ 闭环评测在世界模型仿真器中进行
    ▼
OTA 推送到车队
    └──────────────────────────── 回到开头
```

世界模型插入在两处：**场景生成**（扩充标注数据）和**闭环评测**（替代游戏引擎仿真器）。

### 7.5 数据飞轮（Data Flywheel）

```
更多用户购车
    │
    ▼
更多真实路况数据（含本地特色长尾场景）
    │
    ▼
训练出更好的模型
    │
    ▼
驾驶体验更好、更安全
    │
    ▼
口碑传播 → 更多用户购车 ──→ 循环
```

飞轮的护城河不是模型架构（架构可以复制），而是**特定分布的数据积累**：

- 特斯拉：美国高速场景数据全球最多，但中国城区覆盖差
- 蔚来/小鹏/华为：深圳/上海无保护路口、电动摩托车、占道施工等中国特有场景
- **两个飞轮在不同数据分布上各自转**——这是特斯拉在中国城区 NOA 一度落后于本土方案的根本原因

世界模型在飞轮里是**放大器**：1 份真实数据 × 世界模型 = N 份合成变体，让飞轮在数据量有限时也能转得更快。

## 八、前沿发展（2025–2026）

### 8.1 统一世界模型：从游戏到机器人到自动驾驶

2025 年以来，出现了多个横跨不同领域的通用世界模型研究：

- **Genie 2**（Google DeepMind，2024）：从单张图片生成可交互的 3D 世界，用户可以在生成的世界里走动——不限于特定任务，是迈向"通用世界模型"的尝试
- **DreamDojo**（NVIDIA，2026）：专门解决"如何用大规模人类视频预训练机器人世界模型基础"，是 DreamZero 的伴随工作（见 WAM 专题）

### 8.2 物理引擎 + 生成模型的融合

纯神经网络世界模型的最大弱点是物理不准确。2025 年出现了将经典物理引擎与神经生成模型结合的工作：

- **Genesis**（多机构，2025）：物理仿真引擎 + 生成模型，追求"物理正确"的合成数据，同时保持视觉真实度
- 核心思路：用物理引擎约束生成过程（保证接触力、惯性、摩擦等物理量合理），再用神经网络生成真实感的视觉渲染

### 8.3 世界模型评估体系

随着工作增多，如何评估世界模型的"物理正确性"成为新问题：

- 传统指标（FVD、PSNR）只评估视觉质量，不评估物理一致性
- 新兴评估方向：物体轨迹预测准确率、力/扭矩预测误差、接触点预测精度
- **PhyGenBench**、**RobotWB** 等基准开始出现，专门测试世界模型对物理规律的理解

### 8.4 Test-time Scaling 与世界模型的结合

受 LLM 推理扩展（DeepSeek-R1、o1）启发，将 Test-time Scaling 引入具身控制：在执行前用世界模型搜索多条候选轨迹，奖励模型选出最优路径。这本质是路径二（推理时规划）的神经网络版本，和 WAM 的闭环纠偏方向互补。

> **📖 关联讲义：Week 7 Part 2**，推理时搜索（MCTS、Beam Search + 过程奖励）的通用框架在 Week 7 中有介绍，具身规划可以视为该框架在物理世界中的应用。

## 📎 附录

### A.1 DreamerV3 与 RSSM 简介

> 关联正文：**第三节 → 路径一 Model-based RL**

**DreamerV3**（Hafner et al., 2023）是潜空间世界模型的标杆工作。其核心组件是 **RSSM（Recurrent State Space Model）**：

$$h_t = f(h_{t-1}, s_{t-1}, a_{t-1}) \quad \text{（确定性状态，GRU 递推）}$$
$$s_t \sim p(s_t \mid h_t) \quad \text{（随机状态，编码当前观测的不确定性）}$$

RSSM 将状态分为**确定性部分**（记忆历史的 RNN 隐状态）和**随机部分**（表示当前时刻不确定性的随机变量），策略完全在 $(h_t, s_t)$ 的潜空间里训练，从不见真实像素。

DreamerV3 的成功在于：用同一套超参数在 Atari、连续控制（MuJoCo）、机器人操作等完全不同的任务上均达到当时 SOTA，是少数真正意义上"跨领域统一"的强化学习方法。

### A.2 感知漂移（Perceptual Drift）

> 关联正文：**第三节 → 路径三 视频仿真器**

感知漂移指合成数据与真实数据之间的系统性视觉分布差异，导致在合成数据上训练的模型在真实场景中性能下降。

主要来源：
1. **渲染差距**：游戏引擎的光照/材质与真实物理光学不同
2. **物理细节缺失**：合成视频缺乏真实物体的微小形变、表面纹理变化
3. **接触不真实**：抓取时真实物体的接触面受力形变，仿真里常常没有

神经世界模型（视频生成）比游戏引擎仿真的感知漂移更小，但在**接触力学**和**小物体精细操作**上仍然不准确。这是当前视频仿真器路线的核心技术瓶颈。
