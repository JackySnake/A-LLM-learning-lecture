# 具身智能实践路线：从理论到仿真实践

*整理于完成 VLA / WAM / 世界模型 / 扩散模型 / 自监督表征学习 五个专题学习之后*

> **适用场景**：有 VLA / WAM 理论基础，希望通过仿真实验建立动手经验，但手边没有物理机器人。
>
> **预计周期**：10–18 周（可根据时间灵活调整，Phase 1–3 是核心，Phase 4 是可选研究方向）

---

## 关于"没有物理机器人"

这不是障碍。具身智能研究的主战场本来就在仿真环境里，真实机器人通常只在最后验证阶段出现。

**没有机器人能做的事：**
- 仿真环境训练和评估（LeRobot、LIBERO、SimplerEnv、IsaacLab 全部在电脑上跑）
- 下载并评估预训练模型（OpenVLA、Diffusion Policy、ACT 代码和权重全部开源）
- 使用公开数据集（Open X-Embodiment、BridgeData v2、DROID 可直接下载）
- 仿真演示数据采集（用脚本策略自动生成，或键盘/spacemouse 遥操作）

**没有机器人做不到的事：**
- Sim-to-Real 迁移验证
- 真实场景的鲁棒性测试

大多数顶会论文的核心贡献在仿真里完成，真实机器人实验只是补充验证。

## Phase 1：工具链上手（2–3 周）

### 目标
跑通完整的训练-评估流程，建立对"仿真任务 → 策略训练 → 成功率评估"管线的直觉。

### 工具：LeRobot（Hugging Face）

LeRobot 是目前最适合入门的学习切入点，提供统一的训练/评估管线，支持 ACT、Diffusion Policy 等多种算法，有仿真环境（不需要真实机器人）：

```bash
pip install lerobot
# 支持的仿真环境：
#   gym-pusht   — 推方块进目标区域（2D，最简单）
#   gym-xarm    — xArm 机械臂抓取（3D）
#   gym-aloha   — 双臂 Aloha 精细操作（3D）
```

### 动手任务

**任务 1**：跑通 `gym-pusht` 示例，理解整个流程

```bash
# 生成演示数据（脚本策略自动采集）
python lerobot/scripts/push_dataset_to_hub.py --env gym_pusht

# 训练 ACT 策略
python lerobot/scripts/train.py \
  --policy.type act \
  --env.type gym_pusht \
  --dataset.repo_id lerobot/pusht

# 可视化评估
python lerobot/scripts/eval.py --policy.path outputs/train/...
```

**任务 2**：把算法从 ACT 换成 Diffusion Policy，对比成功率

这直接验证了 VLA 专题第六节的理论——两种动作解码方式在同一仿真任务上的实际差异。预期结论：Diffusion Policy 在精细操作上应优于 ACT（但训练更慢）。

**任务 3**：调整 Action Chunk 大小（ACT 里的 `chunk_size` 参数），观察成功率如何随之变化

对应 VLA 专题第七节 Action Chunking 的理论：窗口太小 → 控制抖动；窗口太大 → 无法及时响应环境变化。

### 检查点
- [ ] 能独立跑通 LeRobot 的完整训练-评估循环
- [ ] 能修改算法配置并对比结果
- [ ] 能看懂 loss 曲线和成功率曲线

## Phase 2：VLA 评估实践（2–3 周）

### 目标
用预训练的大型 VLA 模型做零样本 / 少样本评估，建立对"预训练迁移能力"的直觉。

### 工具：SimplerEnv + OpenVLA

SimplerEnv 专门为评估预训练 VLA 模型设计，在 Google Scanned Objects 上重建了 BridgeData 场景，与真实机器人评估的相关性极高（Spearman r ≈ 0.9）：

```bash
pip install simpler-env
pip install openvla  # 或直接从 HuggingFace 加载权重
```

### 动手任务

**任务 1**：零样本评估 OpenVLA

```python
from transformers import AutoModelForVision2Seq, AutoProcessor

# 加载预训练权重（约 14GB）
model = AutoModelForVision2Seq.from_pretrained(
    "openvla/openvla-7b", torch_dtype=torch.bfloat16
).cuda()

# 在 SimplerEnv 的 put_spoon_on_tablecloth 任务上评估
# 预期零样本成功率：约 40–60%（任务依赖）
```

**任务 2**：prompt 工程实验

修改语言指令描述，观察不同 prompt 写法对成功率的影响：
- `"put the spoon on the cloth"` vs `"place the metal spoon onto the white tablecloth"`
- 直观感受 SigLIP 的语言-视觉对齐效果和局限

**任务 3**：Token 离散化 vs Flow Matching 对比

- OpenVLA（Token 离散化）：官方基线
- OpenVLA-OFT（Flow Matching 动作头）：官方改进版

在同一任务上分别评估，验证 VLA 专题第六节 6.5 节的理论：FM 解码应该在精度上优于 Token 离散化。

### 检查点
- [ ] 能在 SimplerEnv 上加载并评估 OpenVLA
- [ ] 理解任务成功率的评估协议（每个任务运行多少次、如何计算均值）
- [ ] 能解释 Token 离散化和 FM 解码成功率差异背后的原因

## Phase 3：微调实践（3–4 周）

### 目标
在仿真数据上完成一次完整的预训练→微调→评估流程，体验"领域适应"的实际效果。

### 工具：OpenVLA-OFT + LIBERO

LIBERO 提供 4 个标准化任务套件，是目前最常用的 VLA 微调评估基准：

| 套件 | 内容 | 测试目标 |
|------|------|---------|
| LIBERO-Spatial | 同一物体，不同空间位置 | 位置泛化 |
| LIBERO-Object | 同一动作，不同物体 | 物体泛化 |
| LIBERO-Goal | 相同物体，不同目标状态 | 语义泛化 |
| LIBERO-Long | 长时序多步骤任务 | 规划能力 |

### 动手任务

**任务 1**：采集仿真演示数据

LIBERO 自带脚本策略，可以自动生成高质量演示：

```bash
# 生成 LIBERO-Spatial 的 50 条演示（约 30 分钟）
python libero/scripts/collect_demonstrations.py \
  --task-suite LIBERO_SPATIAL \
  --num-demos 50
```

**任务 2**：LoRA 微调 OpenVLA-OFT

```bash
# 官方推荐配置：A100 40GB 或 4090 24GB + LoRA
# 也可以 QLoRA 降到 12GB（3090 可跑）
python train_openvla_oft.py \
  --dataset_path path/to/libero_spatial \
  --lora_rank 32 \
  --batch_size 8 \
  --max_steps 5000
# 预期训练时间：约 2–4 小时（A100）
```

**任务 3**：微调前后成功率对比

在 LIBERO-Spatial 的 10 个任务上分别测试：
- 零样本 OpenVLA-OFT（未微调）：预期约 40–50%
- 微调后 OpenVLA-OFT（50 条演示）：预期约 75–85%

理解"少量演示微调"带来的提升幅度，以及剩余失败案例的模式。

**进阶实验**：在 LIBERO-Long 上对比不同 chunk_size（1 / 4 / 8 / 16），验证 Action Chunking 对长时序任务的作用。

### 硬件要求

| 配置 | 可做的事 |
|------|---------|
| 24GB VRAM（RTX 3090/4090） | OpenVLA-OFT QLoRA 微调、全部评估实验 |
| 16GB VRAM（RTX 3080） | Diffusion Policy / ACT 训练、评估实验，不能微调 7B 模型 |
| 8GB VRAM | 只能跑评估，训练需要 CPU offload（很慢） |
| 云 GPU（Lambda Labs / Colab Pro+） | 按小时租 A100，一次微调实验约 $5–15 |

- [ ] 能完整走一遍"采集数据 → 微调 → 评估"的流程
- [ ] 理解微调带来的提升幅度和局限
- [ ] 能解释某个任务为什么依然失败（是感知问题？规划问题？执行精度问题？）

## Phase 4：研究级实践（可选，4–8 周）

在 Phase 1–3 建立基础之后，可以开始做有研究价值的工作。以下三个方向任选其一：

### 方向 A：复现 + 消融实验

选一篇 2023–2024 年有开源代码的论文，在 LIBERO 上复现结果，再做消融：

**推荐论文**（难度从低到高）：
- **Diffusion Policy**（2023，MIT）：最经典，文档完善，LIBERO 上约 85% 成功率
- **ACT**（2023，Berkeley）：CVAE + Action Chunking，在 LIBERO 上约 90%
- **Octo**（2023，Berkeley）：第一个系统性验证"预训练-微调范式"的开源 VLA

**消融方向举例**：
- 去掉 Action Chunking 后成功率变化
- 不同 Vision Encoder（ResNet vs ViT vs SigLIP）的影响
- 演示数据量从 50 条减少到 10 条时的性能衰退曲线

### 方向 B：IDM 数据效率验证

直接验证 WAM 专题的核心主张："少量演示 + 无标注视频 = 比纯演示更好的策略"。

**实验设计**：
1. 用 IsaacLab 生成大量无动作标签的机器人运动视频（随机策略运动即可）
2. 在这些视频上预训练一个小型 IDM 模型（给定相邻两帧，预测之间发生了什么动作）
3. 用 IDM 从无标注视频中提取伪动作标签
4. 对比：仅 50 条真实演示 vs 50 条真实演示 + IDM 伪标签视频数据
5. 预期：加入伪标签后，成功率应有提升，且在 OOD 场景下提升更明显

**价值**：这是 LingBot-VA 的核心思路在小规模仿真上的验证，结果可以作为研究笔记或技术博客。

### 方向 C：合成数据补充实验

验证世界模型专题"路径三：视频生成仿真器"的实用性。

**实验设计**：
1. 训练一个小型视频预测模型（在 LIBERO 的仿真视频上）
2. 用这个视频模型生成"如果动作稍有变化，未来会是什么样"的合成视频帧
3. 把合成帧作为额外训练数据加入策略训练
4. 评估是否提升了 OOD 泛化（在训练集外的物体/位置上）

**难度**：这是三个方向里最有研究价值但也最难的，建议先完成方向 A 后再挑战。
